プログラムの3つの基本構造

すべてのプログラムは、以下の3つの構造の組み合わせで作られています。

構造意味Python での例
順次（Sequential）上から順番に1行ずつ実行命令を1行ずつ書く
分岐（Selection）条件によって実行する命令を変えるif / else
反復（Iteration）同じ処理を繰り返すfor / while

今回は順次と反復（for文）を中心に学びます。分岐は第3回で扱います。

料理のレシピと同じです。「材料を切る → 炒める → 盛り付ける」は順次。「味が薄ければ塩を足す」は分岐。「5分間かき混ぜ続ける」は反復です。

アルゴリズムと計算量

アルゴリズムとは

「問題を解くための手順」のこと。繰り返し（ループ）は最も基本的なアルゴリズムです。

O(n) ってなに？ ── 計算量の表し方

O(n) は「オーダー・エヌ」と読みます。これは「アルゴリズムの速さを表す記号」で、データの量 n が増えたときに、処理回数（=かかる時間）がどう増えるかを表します。

n（エヌ）とは？ ── 扱うデータの個数です。たとえば：

  • 名簿に載っている学生の人数 → n
  • 画面に描く四角形の数 → n
  • リストの要素数 → n

O(n) の意味 ── 「n が10倍になったら、処理回数もだいたい10倍になる」ということ。n に比例して増えると覚えてOK。

Python
# これは O(n) ── n回ループするだけ
for i in range(n):
    print(i)

なぜ「だいたい」でいいの？ ── 細かい定数（+5回とか ×2倍とか）は無視して、増え方の「形」だけに注目するのが O 記法の考え方です。n が大きくなればなるほど、定数の差は誤差レベルになるからです。

記号読み方増え方のイメージ例
O(1)オーダー・1n が増えても変わらないリストの先頭を取り出す
O(n)オーダー・エヌn に比例して増える1重ループ
O(n2)オーダー・エヌ二乗n の二乗で爆発的に増える2重ループ
O(log n)オーダー・ログnn が増えても少ししか増えない二分探索
O(n) = 「教室の全員に1回ずつ出席を取る」── 30人なら30回、300人なら300回。人数に比例して時間がかかる。
O(n2) = 「全員が全員と握手する」── 30人なら900回、300人なら90,000回。人数の二乗で増える。

※「O」は Order（オーダー、増え方の「位」）の頭文字。アルゴリズムの良し悪しを比較するときの共通言語です。

計算量の直感

ループの回数が増えると処理時間が増えます。特に二重ループは要注意です。

ループn=10 の処理回数n=100 の処理回数n=1000 の処理回数
1重ループ（O(n)）10回100回1,000回
2重ループ（O(n2)）100回10,000回1,000,000回

step2-5.py の二重ループで体感してみましょう。range の数を変えると描画時間がどう変わるか試してみてください。

1重ループは「教室の全員に1回ずつ握手する」（n回）。2重ループは「全員が全員と握手する」（n×n回）。人数が10人なら100回、100人なら10,000回になります。

図の読み方

左の1重ループは横一列の5マス（5回の処理）。右の2重ループは外側ループ i が1回進むたびに、内側ループ j が0→4まで5回回ります。つまり i の各行に5マスずつ処理があり、合計で 5×5 = 25マス。これが n² の意味です。

step2-5.py の二重ループはまさにこの構造で、縦5×横5＝25個の四角形を描いています。range の数字を10にすると100個、50にすると2,500個になり、描画が遅くなるのが体感できます。

O(n²) は「指数関数」ではない ── 念のため

このグラフの形（放物線）は「多項式（=2乗）」の増え方です。よく似た言葉に指数関数 O(2n) がありますが、これは別物です：

  • O(n²): n が10倍 → 処理は100倍（n×n）
  • O(2n): n が1増えるごとに処理が2倍。n=100 で 約 1030 回 ── 宇宙の年齢でも終わらない

2重ループは「ヤバいけど対処可能」、指数時間は「nが30を超えたら現実的に解けない」── このランクの違いを覚えておきましょう。

入れ子（ネスト）for文の書き方 ── 2重・3重ループ

for文の中にfor文を書くと「入れ子（ネスト）」になります。インデント（字下げ）が深くなるほどループが深くなります。

Python
# 2重ループ ── i × j の組み合わせを全部出す（3×3 = 9回）
for i in range(3):
    for j in range(3):
        print(f"i={i}, j={j}")

出力（一部）:

Output
i=0, j=0
i=0, j=1
i=0, j=2
i=1, j=0
...
i=2, j=2
Python
# 3重ループ ── i × j × k の組み合わせ（2×2×2 = 8回）
for i in range(2):
    for j in range(2):
        for k in range(2):
            print(f"i={i}, j={j}, k={k}")
時計の「時・分・秒」のイメージ。外側のループ（時）が1進むあいだに真ん中（分）が60回、内側（秒）はさらに60回ずつ。3重ループの実行回数は「外×中×内」の掛け算になります。

体験 ── 処理時間を測ってみよう

「2重ループ・3重ループは本当に遅いのか？」を、Python標準の time モジュールで実測してみましょう。

Python (time_loops.py)
import time

n = 1000

# O(n) ── 1重ループ: 1,000回
start = time.perf_counter()
for i in range(n):
    pass
print(f"O(n)   1重ループ n={n}: {time.perf_counter() - start:.4f}秒")

# O(n²) ── 2重ループ: 1,000,000回（100万回）
start = time.perf_counter()
for i in range(n):
    for j in range(n):
        pass
print(f"O(n²)  2重ループ n={n}: {time.perf_counter() - start:.4f}秒")

# O(n³) ── 3重ループ: 1,000,000,000回（10億回）── 数十秒〜1分以上かかる！
start = time.perf_counter()
for i in range(n):
    for j in range(n):
        for k in range(n):
            pass
print(f"O(n³)  3重ループ n={n}: {time.perf_counter() - start:.4f}秒")

実行例（環境により異なります）:

Output
O(n)   1重ループ n=1000:    0.0000秒    # 一瞬
O(n²)  2重ループ n=1000:    0.0300秒    # まだ速い
O(n³)  3重ループ n=1000:   45.2000秒    # 数十秒待たされる！

注意 ── 3重ループは数十秒待つことを覚悟

n=1000 で3重ループを回すと 10億回（109）。PCの性能にもよりますが 30秒〜1分以上かかります。「フリーズした？」と思ってもターミナルやエディタを閉じず、そのまま待つのがコツ。あまりに遅すぎる場合は Ctrl + C で中断できます（このまさに「待たされる感覚」が O(n³) の本質です）。

データの表現と種類

色 = 数値3つで表現

コンピュータでは、色を3つの数値の組み合わせで表現します。すべての情報は数値化できる ── これがデータサイエンスの出発点です。

  • RGB: 赤(R)・緑(G)・青(B) の光の強さ（各 0〜255）
  • HSB: 色相(H)・彩度(S)・明度(B) で色を指定

そもそも「データ」とは？ ── データ・情報・知識の違い

「データの種類」を学ぶ前に、データ・情報・知識の関係を整理しましょう。これは情報学の基本概念で、DIKWピラミッドとも呼ばれます。

段階定義（穴埋め式の言い方）例
データ測定・記録できるもの（生の事実、まだ意味なし）36.5、A型、晴れ
情報人間が認識できたもの（データに文脈・意味を付けた）「今朝の体温は 36.5℃」「彼の血液型は A型」
知識判断・行動に使えるもの（情報を経験・法則と結びつけた）「37℃ 以上が続いたら病院へ」「A型は几帳面な傾向」
料理で例えると ── データ は「冷蔵庫の中の食材（生のニンジン、玉ねぎ、肉）」、情報 は「これらを使うとカレーが作れる、と気づくこと」、知識 は「お客さんに合わせて辛さを調整する経験則」。生の食材だけでは料理にならないし、レシピだけでは美味しくならない。

各段階で「漏れ」がある ── ピラミッドが逆三角形なのはそのため

世界にはまだデータ化できていないものがたくさんあります。職人の勘、表情の微妙な変化、深海や宇宙の未観測領域、絶滅した生物の生態 ── こうしたものは数値にも記号にもなっていません。さらに、せっかくデータ化されても 人間が解釈できないもの（ノイズだらけのセンサー値、膨大すぎて読めないログ）や、情報になっても活かしきれないもの（書かれているのに誰も読まないレポート）が大量に存在します。

図の左側に出ている赤い矢印が、それぞれの段階で「外に漏れていくもの」を表しています。データサイエンスとは、この漏れをいかに減らして、世界の現象を行動可能な知識に変換するかという挑戦です。

この授業で扱う print(36.5) や CSV ファイルの中身は 「データ」。それを Python で集計・可視化して人が理解できる形にすると 「情報」。さらに「だから次はこうしよう」と意思決定に繋げると 「知識」になります。そして、まだ誰もデータ化していない現象を新しく測定できるようにすることも、データサイエンスの大事な仕事です。

データの種類 ── 種類によって「できる処理」と「向いてるグラフ」が違う

データの種類は4つに分かれ、それぞれ 使える計算（統計量） と 適切なグラフ が決まっています。種類を間違えると、平均0.5血液型のような意味のない計算をしてしまいます。

離散

なぜ種類で分けるの？ ── やってはいけない計算がある

  • 名義尺度に平均を取るな: 「A型・B型・O型・AB型の平均」に意味はない。カウントと比率だけが使える
  • 順序尺度の平均は要注意: 「満足度1〜5」の平均が3.7でも、満足度2と3の差と、4と5の差が等しい保証はない
  • 量的データはほぼ何でもOK: 平均・分散・相関と、強力な統計手法が全部使える
  • グラフ選び: 質的データに 折れ線グラフはNG（点と点を線で繋ぐ意味がない）。連続データに 円グラフはNG（カテゴリじゃない）

色のRGB値（0〜255の整数）は離散的な量的データ。だから RGB の平均を取って色を混ぜることもできるし、ヒストグラムで色の分布を出すこともできる ── これが第3回以降の画像処理の出発点です。

色の表現 RGB・HSB ── 約30分

RGBとHSBの違い

RGB（Red, Green, Blue）

光の三原色。各色 0〜255 で指定します。py5のデフォルトのカラーモードです。

色RGB
赤25500
緑02550
青00255
黄2552550
白255255255
黒000

HSB（Hue, Saturation, Brightness）

人間の感覚に近い色指定方法。「虹のどの色か」「どれくらい鮮やかか」「どれくらい明るいか」で指定します。

パラメータ範囲意味
H（色相）0〜360色の種類（0=赤, 60=黄, 120=緑, 180=シアン, 240=青, 300=紫）
S（彩度）0〜100鮮やかさ（0=灰色, 100=最も鮮やか）
B（明度）0〜100明るさ（0=黒, 100=最も明るい）

HSBの利点: 「虹色のグラデーション」を作りたいとき、Hの値を変えるだけでよい。RGBでは3つの値を同時に調整する必要があり複雑。

実験1

RGB vs HSB の比較

以下のコードを実行して、RGBとHSBで同じ「赤」を表現できることを確認してください。

Python ── experiment1.py
# 実験1 RGB vs HSB comparison
import py5

def setup():
    py5.size(400, 200)
    py5.background(255)
    py5.no_stroke()

    # RGB モード（デフォルト）
    py5.color_mode(py5.RGB, 255)
    py5.fill(255, 0, 0)
    py5.circle(100, 100, 80)

    # HSB モード
    py5.color_mode(py5.HSB, 360, 100, 100)
    py5.fill(0, 100, 100)
    py5.circle(300, 100, 80)

py5.run_sketch()

確認ポイント

  • 左の円（RGB: 255, 0, 0）と右の円（HSB: 0, 100, 100）は同じ赤色になる
  • color_mode でカラーモードを切り替えられる
  • HSBの H=0 は赤（色相環の開始位置）

発展課題は任意です。標準課題（std2-1〜std2-4）を完了してから取り組んでください。提出は コード＋実行結果＋音声説明（1〜2分） の3点セットを Notion に記録します。音声説明は教員・SAが内容を確認するため、「何を作ったか」だけでなく「どう作ったか／なぜそうしたか」を含めてください。

発展課題

adv2-1: オリジナル作品（自由テーマ） 音声説明必須

テーマは完全に自由。今回学んだ技術（for文 / 二重ループ / RGB / HSB / draw() / マウス入力）を組み合わせて、自分なりの作品を制作してください。std2-4（自由制作）よりも1段難しいことに挑戦するのがおすすめです。

作品例 ── 回転対称模様（rotationally symmetric）

中心からの角度（θ）と半径（r）を二重 for で回し、各点を sin/cos で配置しつつ HSB で色相を変化。draw() で frame_count を使い回転アニメーション化したサンプルです。

※ これは1つの例に過ぎません。同じ模様をコピーするのではなく、これを参考に「自分のオリジナル」を作りましょう。

挑戦例

アイデア使う技術
マウスで色が変わる花畑draw() + mouse_x/y + 二重 for + HSB
キャラクターのアニメーションdraw() + frame_count で位置・色を変化
万華鏡（角度回転）二重 for + 三角関数 (sin, cos) + HSB
ランダムスターrandom + for + 不透明度（α）
波打つグラデーションdraw() + frame_count + sin 波 + HSB

補講ページ ── さらに深く学ぶ

第 2.5 回 ── for 文を徹底的に学ぶ

第 2 回で触れた for 文を、5 つの図形課題 + 10 の小問 + 5 つの練習問題で徹底的に身につけます。

  • range() の仕組みと変数 i の使い方
  • 等間隔配置・グラデーション・同心円など「絵を描くための for 文」
  • 二重ループ（ネスト）でグリッド状に図形を配置する方法

第 2.75 回 ── translate / push / pop を理解する

座標系を操作して「原点を動かして描く」テクニックを、3 つの問題 + 8 つの小問 + 3 つの練習問題で学びます。

  • translate(x, y) で原点を移動し、ローカル座標で描画する方法
  • push() / pop() で座標系をセーブ＆ロードする仕組み
  • rotate() との組み合わせで時計の目盛りや太陽系を描く応用
