条件分岐処理の基本（約20分）

if / elif / else 文

条件分岐とは

条件によって処理を分けることを条件分岐といいます。Pythonでは if、elif、else を使います。

信号機と同じです。「青なら進む、黄色なら止まる準備、赤なら止まる」。条件に応じて行動が変わります。 
Python ── if/elif/else の基本
score = 75

if score >= 90:
    print("優秀です！")
elif score >= 60:
    print("合格です")
else:
    print("不合格です")

インデント（字下げ）が重要

Pythonではインデント（半角スペース4つ）で処理のブロックを表します。インデントがずれるとエラーになります。

比較演算子

演算子意味例結果
==等しい5 == 5True
!=等しくない5 != 3True
>より大きい5 > 3True
<より小さい3 < 5True
>=以上5 >= 5True
<=以下3 <= 5True

== と = を間違えないこと

= は代入、== は比較です。if x = 5: はエラーになります。正しくは if x == 5: です。

論理演算子

演算子意味例
andかつ（両方True）x > 0 and x < 10
orまたは（どちらかTrue）x < 0 or x > 100
not否定（TrueをFalseに）not (x == 0)
Python ── 論理演算子の例
age = 20
has_ticket = True

if age >= 18 and has_ticket:
    print("入場できます")
elif age < 18:
    print("18歳未満は入場不可")
else:
    print("チケットが必要です")

py5で条件分岐活用 ─ アニメーション（約20分）

アクティブモード: setup() + draw()

スタティックモード vs アクティブモード

  • スタティックモード: setup() だけ。1回だけ描画して終了（前回まで）
  • アクティブモード: setup() + draw()。draw() が毎フレーム繰り返し実行される
パラパラ漫画と同じです。draw() が1ページ分の絵を描き、毎秒60回（60fps）新しいページを描くことで、動いているように見えます。 
Python ── アクティブモードの基本構造
import py5

def setup():
    # 最初に1回だけ実行される
    py5.size(800, 400)
    py5.frame_rate(60)  # 1秒間に60回 draw() を呼ぶ

def draw():
    # 毎フレーム繰り返し実行される
    py5.background(192)  # 毎回背景をリセット
    py5.ellipse(py5.mouse_x, py5.mouse_y, 50, 50)

py5.run_sketch()

グローバル変数と global 宣言

なぜ global が必要なのか

関数の中で定義した変数はローカル変数で、その関数の中でしか使えません。setup() で作った変数を draw() で使うには、関数の外（グローバルスコープ）で変数を定義し、関数内で global 宣言をして「この変数はグローバル変数ですよ」と伝えます。

Python ── global 宣言の例
import py5

x = 0      # グローバル変数
dx = 5     # 移動速度

def setup():
    py5.size(800, 400)
    py5.fill(0, 128, 255)
    py5.frame_rate(60)

def draw():
    global x, dx          # グローバル変数を変更するために必要
    py5.background(192)
    py5.ellipse(x, 200, 50, 50)
    x += dx               # 毎フレーム x を dx だけ移動

py5.run_sketch()

global を忘れるとどうなる？

global を書かずに x += dx とすると、Python は x をローカル変数だと解釈し、UnboundLocalError が発生します。

練習: ボールの跳ね返り（左右）

ノート

上のコードでは、ボールが画面右端を超えて消えてしまいます。条件分岐を使って、端に到達したら方向を反転させましょう。

Python ── 跳ね返りの追加
import py5

x = 0
dx = 5

def setup():
    py5.size(800, 400)
    py5.fill(0, 128, 255)
    py5.frame_rate(60)

def draw():
    global x, dx
    py5.background(192)
    py5.ellipse(x, 200, 50, 50)
    x += dx

    # 端に到達したら方向を反転
    if x > py5.width or x < 0:
        dx = -dx

py5.run_sketch()

なぜ跳ね返るのか？

dx = -dx で速度の正負を反転しています。dx = 5 が dx = -5 になると、x座標が減少する＝左に移動します。再び左端に到達すると dx = 5 に戻り、右に移動します。

練習: 色チェンジャ

ノート

キーボードの r、g、b キーを押すと円の色が変わるプログラムです。

Python ── color_changer.py
import py5

r = 128
g = 128
b = 128

def setup():
    py5.size(400, 400)

def draw():
    py5.background(220)
    py5.fill(r, g, b)
    py5.ellipse(200, 200, 150, 150)

def key_pressed():
    global r, g, b
    if py5.key == 'r':
        r, g, b = 255, 0, 0
    elif py5.key == 'g':
        r, g, b = 0, 255, 0
    elif py5.key == 'b':
        r, g, b = 0, 0, 255

py5.run_sketch()

拡張してみよう

マウスクリックでも色を変えたい場合、mouse_pressed() 関数を追加しましょう。py5.mouse_button == py5.LEFT で左クリック、py5.mouse_button == py5.RIGHT で右クリックを判定できます。

CS/DS 基礎知識

状態遷移（State Transition）

ボールの跳ね返り＝状態機械の実例

ボールの動きは「右に移動中」「左に移動中」という状態（State）を持っています。壁に当たると状態が切り替わります。これを状態機械（State Machine）といい、ゲームやUIの設計で広く使われる概念です。

自動販売機と同じです。「待機中」→ お金投入 →「選択待ち」→ ボタン押す →「払い出し中」→「待機中」と状態が遷移します。

CPUはif文をどう処理するか

条件分岐 ＝ CPUの基本動作

CPUは条件分岐命令（branch instruction）で「次にどの命令を実行するか」を切り替えます。Pythonの if 文は最終的にCPUの条件分岐命令に変換されます。

  • 比較（CMP）: 2つの値を比較してフラグをセット
  • 分岐（JMP/BEQ/BNE）: フラグに応じて実行する命令のアドレスを変更

プロセスとスレッド

概念の理解

  • プロセス: 実行中のプログラム。独自のメモリ空間を持つ。py5のスケッチは1つのプロセスとして動く
  • スレッド: プロセスの中の実行単位。1つのプロセスに複数のスレッドを持てる。py5の draw() は描画スレッドで実行される
会社（プロセス）と社員（スレッド）の関係です。会社はオフィス（メモリ）を持ち、社員はそのオフィスの中で同時に仕事をします。

スコープ（ローカル変数とグローバル変数）

変数の有効範囲

  • ローカル変数: 関数の中で定義した変数。その関数内でのみ有効
  • グローバル変数: 関数の外で定義した変数。どこからでも参照可能（変更には global 宣言が必要）
Python ── スコープの例
message = "こんにちは"  # グローバル変数

def greet():
    name = "太郎"       # ローカル変数（greet の中だけで有効）
    print(message, name)  # グローバル変数は参照可能

greet()          # → こんにちは 太郎
print(message)  # → こんにちは
# print(name)  → NameError（ローカル変数は外から参照不可）

関数（引数と戻り値）

関数の基本

関数は処理をまとめたものです。引数で値を受け取り、戻り値で結果を返します。

Python ── 関数の例
def add(a, b):        # a, b は引数（パラメータ）
    return a + b       # 戻り値を返す

result = add(3, 5)    # 引数に 3 と 5 を渡す
print(result)           # → 8
電子レンジ（関数）と同じです。食材（引数）を入れて、温かい料理（戻り値）が出てきます。電子レンジの中身（処理の詳細）を知らなくても使えます。

練習2: FizzBuzz ─ 解答例

「両方の倍数」を最初に判定する理由: 15 は3の倍数でもあるので、if n % 3 == 0 を先に書くと "Fizz" が表示されてしまい、"FizzBuzz" にたどり着きません。

Python ── FizzBuzz 解答
n = 15

if n % 3 == 0 and n % 5 == 0:
    print("FizzBuzz")
elif n % 3 == 0:
    print("Fizz")
elif n % 5 == 0:
    print("Buzz")
else:
    print(n)
std3-1: ボールの跳ね返り ─ 解答例

ベースコードに上下キーの処理と、上下左右すべての端での跳ね返りを追加したコードです。

Python ── std3-1 解答
import py5

x, y = 200, 150
dx, dy = 0, 0
speed = 5

def setup():
    py5.size(400, 300)

def draw():
    global x, y, dx, dy
    py5.background(220)
    x += dx
    y += dy

    # 左右の跳ね返り
    if x > py5.width or x < 0:
        dx = -dx
    # 上下の跳ね返り
    if y > py5.height or y < 0:
        dy = -dy

    py5.circle(x, y, 30)

def key_pressed():
    global dx, dy
    if py5.key_code == py5.LEFT:
        dx, dy = -speed, 0
    elif py5.key_code == py5.RIGHT:
        dx, dy = speed, 0
    elif py5.key_code == py5.UP:
        dx, dy = 0, -speed
    elif py5.key_code == py5.DOWN:
        dx, dy = 0, speed

py5.run_sketch()

ポイント

  • draw() 内で dx と dy も global 宣言に追加（跳ね返りで値を変更するため）
  • 左右と上下の跳ね返りは別々の if 文で処理（elif ではない。x方向とy方向は独立に判定するため）
  • ボールの半径を考慮する場合は x > py5.width - 15 のように調整する
