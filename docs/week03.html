条件分岐 + アニメーション + キー入力

ノート

① 条件分岐 ── if / elif / else

仕組み: 「もし○○ならA、そうでなければB」というふうに、値によって動く処理を切り替えること。Pythonでは if 文で書きます。

▼ まずは if 文の「型」を覚えよう ── キーワード／コロン／インデント

if 文は 「キーワード + 条件 + コロン (:) + 改行 + インデントされた処理」 という決まった形をしています。コロンとインデントを忘れるとエラーになるので、最初に型を覚えましょう。

番号部品意味・ルール
①if キーワード「もし〜なら」の合図。必ず行頭に書く（インデントなし）
②条件式True または False を返す式。score >= 90 や x == 0 など
③:（コロン）必須。条件式の終わりに半角コロン。忘れると SyntaxError
④インデント半角スペース 4 個で字下げ。同じブロック内は同じ深さ。忘れると IndentationError
⑤elifelse if の略。前の if/elif がすべて False のとき次の条件を評価。何個でも繋げてOK（0個でも可）
⑥else（任意）どの条件にも当てはまらなかった場合。条件式は書かない（else: のみ）。省略してもよい

❌ コロンを忘れた例

if score >= 90      # : がない
    print("優秀")

→ SyntaxError になる

❌ インデントを忘れた例

if score >= 90:
print("優秀")            # 字下げなし

→ IndentationError になる

ポイント: if 文は「ブロック」を作る

if 〜 : の次の行から 同じインデント幅で書かれた処理のかたまりが「if のブロック」になります。インデントを戻すと if のブロックは終わりです。

条件なし(常に同じ動作)

Python
score = 75
print("優秀!")
# scoreが何でも
# 必ず「優秀!」

条件あり(if文 ─ 値で変わる)

Python
score = 75
if score >= 90:
    print("優秀!")
else:
    print("普通")

② アニメーション ── setup() + draw()

仕組み: py5は1秒間に約60回 draw() を呼び続けます。draw() の中で図形の位置や色を少しずつ変えると、人間の目には動いて見えます。

▼ アニメーションの原理 ── パラパラ漫画と同じ「ずらして見せる」方式

1秒間に60枚の絵を、少しずつ位置をずらしながらパッパッと切り替えて表示する。それだけで人間の目には滑らかに動いて見える。これがアニメーションの正体です。

📺 テレビや映画と同じ仕組み

テレビは 1秒間に 30〜60 枚、映画は 24 枚の静止画を切り替えて「動いて見せて」います。py5 は 60 枚 / 秒。

📖 パラパラ漫画と同じ

紙の端に少しずつ違う絵を描いてめくる「パラパラ漫画」と原理は同じ。人間の目は「残像」で滑らかに見せてくれる（残像効果）。

py5 における「ずらして見せる」の正体

py5 では、draw() 関数を 1秒間に 60 回呼んでくれます。私たちは draw() の中で「変数 x をちょっとずつ増やす」だけ書けば OK。すると毎フレーム x が増えた位置に円が描かれ、結果的に動いて見えるのです。
⇒ 次のセクションで x += 2 を毎フレーム実行する例を見ていきます。

▼ 正式な用語: フレームレート（Frame Rate）／ FPS（Frames Per Second）

フレームレートとは 1 秒間に何枚の絵（フレーム）を表示するかを表す数値で、FPS（Frames Per Second）という単位で表します。

  • FPS が大きいほど 1秒間に多くの絵を切り替える → 滑らかに動いて見える
  • FPS が小さいと切り替え回数が少ない → カクカクして見える（コマ送り）
  • py5 のデフォルトは 60 FPS（py5.frame_rate(60) で変更可能）

▼ まず大事: py5キャンバスの座標系（x と y の向き）

py5 では 左上が原点 (0, 0)。x は右に行くほど大きく、y は下に行くほど大きくなります（数学のグラフと違って y は下が +）。

x が増える → 右へ移動
例: x += 2 で円が右へ進む
y が増える → 下へ移動
例: y += 2 で円が下へ進む
y が減る → 上へ移動
例: y -= 2 で円が上へ進む（注意: y は下が +）
x が減る → 左へ移動
例: x -= 2 で円が左へ進む

▼ x += 2 や y -= 2 ってどういう意味？

+= と -= は 「自分自身に値を足す（引く）」を短く書く ための記号です。次のように同じ意味のコードの省略形になります。

x += 2 の意味

# 短い書き方
x += 2

# これと完全に同じ意味
x = x + 2

「今の x の値に 2 を足して、結果を新しい x にする」

5050 + 252
5252 + 254
5454 + 256

y -= 2 の意味

# 短い書き方
y -= 2

# これと完全に同じ意味
y = y - 2

「今の y の値から 2 を引いて、結果を新しい y にする」

実行前 y計算実行後 y
100100 - 298
9898 - 296
9696 - 294

ポイント: = は「等しい」ではなく「代入」

x = x + 2 という式は数学では矛盾していますが、Python では 「右辺を計算して左辺に代入する」という意味です。だから「いまの x に 2 を足したものを、新しい x にする」が成立します。

貯金通帳と同じです。残高 += 1000 は「いまの残高に 1000 円を足して、新しい残高にする」。何度実行しても、そのつど現在の残高を起点に 1000 円ずつ増えていきます。
他の複合代入演算子（参考）
記号意味例同じ意味の書き方
+=足して代入x += 2x = x + 2
-=引いて代入y -= 2y = y - 2
*=掛けて代入x *= 2x = x * 2
/=割って代入x /= 2x = x / 2

アニメーションで += が大事な理由

draw() は毎フレーム呼ばれます。x += 2 と書いておくと、毎フレームごとに「今の x」に 2 を足すので、x は 50 → 52 → 54 → 56 → ... と増え続けます。これが「円が右へ動く」アニメーションの正体です。
もし x = x + 2 でなく x = 2 と書いたら、毎フレーム x が必ず 2 にリセットされてしまい、円は動きません。

▼ cs3-1: setup() / draw() の裏側 ── プロセスとスレッド (CS基礎・Notionに記録)

draw() が「1秒間に約60回」呼ばれ続けるのは、py5 が裏で専用のスレッドを使っているからです。ここでは、その仕組みを優しく解説します。

プロセス（Process）とスレッド（Thread）とは

  • プロセス: OS が 1 つのプログラムを動かす単位。独自のメモリ空間を持ち、他のプロセスから干渉されない。python my_sketch.py を実行すると 1 つの Python プロセスが立ち上がる。
  • スレッド: プロセスの中の 並行に動く実行の流れ。同じプロセス内のメモリ（変数）を共有する。1つのプロセスは複数のスレッドを持てる。
会社（プロセス）と社員（スレッド）の関係です。会社はオフィス（メモリ）を持ち、社員（スレッド）はそのオフィスの中で同時並行に仕事をします。社員同士は同じ書類棚（メモリ＝変数）を見られます。別の会社（別プロセス）の書類棚は見られません。
py5 のスケッチでは何が起きているか

▶ 1 つのプロセスの中に 3 つのスレッドが同時に動く。すべて同じメモリ（変数 x など）を見るので、key_pressed() で書き換えた x が次の draw() ですぐに反映される。

なぜこの仕組みになっているのか？
  • 描画は止まってはいけない: ユーザ入力を待つ間も、画面は描き続けないと「フリーズ」したように見える。だから draw() 専用のスレッドが独立して動く。
  • 入力もリアルタイムに反応したい: キー押下を「待ち受ける」専用スレッドがあれば、draw() の合間でも入力を取り逃さない。
  • 同じ変数を共有できる: プロセス内なのでメモリ共有が自動。global x で書き換えた値が他スレッドからも即座に見える。

setup() と draw() の役割分担

関数呼ばれるスレッド呼ばれるタイミング主な役割
setup()メインスレッド起動時に1回キャンバスサイズ設定、初期化
draw()描画スレッド約60回/秒で繰り返し毎フレームの描画・状態更新
key_pressed()イベントスレッドキー押下の瞬間入力に応じた状態変更
mouse_pressed()イベントスレッドクリックの瞬間マウス操作への反応

初学者の方へ

「複数スレッドが同時に動く」というのは少し難しい概念ですが、py5 では基本的に意識しなくて大丈夫です。global を使えば自動的に共有メモリに書き込まれ、他スレッドからも見えます。「draw() の裏で別の流れが回っている」というイメージだけ持っておけば、key_pressed() や mouse_pressed() がなぜ draw() の途中で割り込むように見えるのかが理解しやすくなります。

CS基礎

「状態」とは何か（一言で）

状態（State）とは、そのプログラム（やモノ）が今とっている「ありかた」「モード」のことです。時間とともに状態は切り替わることができ、その切り替わりを 状態遷移（State Transition） といいます。

📝 ex3-1: 信号機（条件分岐）の解答例
  1. color = "red" → if が True → 「止まる」と出力
  2. color = "yellow" → if は False、elif が True → 「注意」と出力
  3. color = "green" → if/elif どちらも False → else が実行 → 「進む」と出力
  4. color = "purple" → if/elif どちらも False → else が実行 → 「進む」と出力（信号機にない色でも、red/yellow 以外はすべて else に流れ込むため）
  5. 順序入れ替え: red と yellow を交換しても、上から順に判定されるが、color が "red" なら "yellow" のチェックには進まないので結果は同じ。ただし「両方の条件に当てはまる」ケースがある場合は順序が結果を変える（今回は排他的なので影響なし）
  6. else を消した場合: red/yellow 以外（green, purple, "" など）のときは何も出力されない。エラーにはならず、プログラムは何事もなかったかのように終わる
📝 ex3-2: 円が右に動く（アニメーション）の解答例
  1. setup() は 1 回だけ実行される（プログラム開始時）
  2. draw() は 1 秒間に約 60 回呼ばれる（py5 のデフォルトフレームレートが 60 FPS）
  3. 1 秒で x はおよそ 120 増える（60 回 × +2 ＝ 120）
  4. x += 2 を x = 2 に書き換えると、毎フレーム x が 2 にリセットされる。円は (2, 100) に静止して動かない
  5. global x を消すと、x += 2 はローカル変数の代入として解釈されるが初期値がないため UnboundLocalError が出る
  6. x が 400 を超えても draw() は止まらず、x は増え続ける。円は画面外（描画範囲の右側）に進み続けるため見えなくなる
  7. background(220) を消すと: 毎フレームの円が消されずに残り続ける。結果として軌跡（線）として描画される
📝 ex3-3: 矢印キーで円を動かす（ユーザ入力）の解答例
  1. 何もキーを押さないとき、key_pressed() は呼ばれず、x は 200 のまま。円は (200, 100) に静止（draw() は呼ばれ続けるが、毎フレーム同じ場所に描画される）
  2. ← を 1 回押すと x は -20 変化（200 → 180）。key_pressed() は押した瞬間に 1 回だけ呼ばれる
  3. → を 5 回連続で押すと、x は +100 増える（5 × 20 ＝ 100）
  4. py5.LEFT／py5.RIGHT は特殊キーを表す整数定数。文字キー（'a' など）の判定には py5.key（文字列）を使い、矢印・Enter・Shift などの特殊キーには py5.key_code（整数定数）を使う
  5. key_pressed() 関数自体を消すと、キーを押しても何も反応しない。円はずっと (200, 100) に静止したまま
  6. global x を消すと、x -= 20 がローカル変数 x への代入として解釈される。グローバルの x は変わらず、次の draw() でも x=200 のまま。円は動かない（実行時エラーは出ない、論理エラー）
  7. x が負（例: -20）になっても、draw() は circle(-20, 100, 40) を描画する。円は画面の左外に描かれるため見えなくなる（描画自体はされている）
