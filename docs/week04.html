〇〇

ヒント: 文字列は "〇" * 3 のように掛け算で繰り返せる（結果は "〇〇〇"）。for 文を使うと、各行の〇の個数を 3 → 2 → 1 と変えられる。

解答例（1）: いちばんシンプル ── 3回 print するだけ

Python（インタラクティブモード）
>>> print("〇〇〇")
〇〇〇
>>> print("〇〇")
〇〇
>>> print("〇")
〇

ベタ書きだけど、これでも答えとしては正解。ただし行数が増えると大変。

解答例（2）: range() で繰り返す

Python（インタラクティブモード）
>>> for row in range(3):
...     print("〇" * (3 - row))
...
〇〇〇
〇〇
〇

range(3) は 0, 1, 2 を生成。3 - row で個数を 3, 2, 1 と変える。文字列 × 整数で "〇" * 3 == "〇〇〇"。

派生: 積み上げる方向も簡単 ── print("〇" * i) で同じパターンの「逆向き」が作れる

Python（インタラクティブモード）
>>> for i in range(4):
...     print("〇" * i)
...

〇
〇〇
〇〇〇

range(4) は 0, 1, 2, 3 を生成。i をそのまま個数に使うと 0, 1, 2, 3 個の〇が積み上がる（最初の i=0 は空行）。このループ＋掛け算のパターンが分かれば、ピラミッド・階段・グラフのバーなど色々作れる。

問題のゴール（減る向き）に揃えたいなら range(3, 0, -1) で 3, 2, 1 を作る手もある:

Python（インタラクティブモード）
>>> for i in range(3, 0, -1):
...     print("〇" * i)
...
〇〇〇
〇〇
〇

でも、もっと楽に書ける ── リストを使うと便利！

各行の個数 3, 2, 1 を計算で出すのではなく、最初からリストに書いておくと、コードがそのまま「設計図」になる:

Python（インタラクティブモード）── 今日の本命
>>> for n in [3, 2, 1]:
...     print("〇" * n)
...
〇〇〇
〇〇
〇

この [3, 2, 1] がまさに今日のテーマ「リスト」。

  • 形を変えたい？ → リストを書き換えるだけ: [1, 3, 5, 3, 1] でひし形、[2, 4, 6] で逆三角形
  • 計算で個数を出さなくていい ── 設計と実装が分離する
  • あとから読んだとき「3, 2, 1個並べたいんだな」が一目で分かる

→ こうやってデータを「リストにまとめる」のが今日学ぶこと。

変数が「1つの箱」だとすると、リストは「引き出しが並んだ棚」、辞書は「見出し付きの辞典」、関数は「レシピ（材料を入れると料理が出てくる）」です。

CS/DS基礎: データ構造と関数

データ構造とは

プログラムで扱うデータの整理方法のことです。目的に応じて適切なデータ構造を選ぶことで、コードがシンプルで効率的になります。

データ構造特徴日常の例Python
リスト順番のあるデータの集合買い物リスト、出席番号[1, 2, 3]
辞書キーと値のペア英和辞典、電話帳{"名前": "太郎"}

関数とは

処理のまとまりに名前を付けたものです。引数（材料）を受け取り、処理（調理）をして、戻り値（料理）を返します。

引数（入力） → 関数（処理） → 戻り値（出力）

なぜ関数に分割するか

  • 再利用: 同じ処理を何度も書かなくてよい
  • 可読性: コードに名前がつくので読みやすい
  • テスト容易性: 小さい単位でテストしやすい
  • 修正容易性: 1か所直せば全体に反映される

リスト（list）とは

複数のデータを順番に並べて1つの変数で管理できるデータ構造です。角カッコ [ ] で囲み、カンマ , で区切ります。

リストは「ロッカーの列」です。0番、1番、2番...と番号（インデックス）がついていて、番号を指定して中身を取り出せます。
リスト (list)順番のあるデータの集合。[ ]で作る インデックスリスト内の位置番号。0から始まる append()リスト末尾に要素を追加するメソッド スライスlist[1:3] で範囲を切り出す操作

リストの基本操作

1. リストの作成

Python
# リストの作成
fruits = ["りんご", "バナナ", "みかん"]
numbers = [10, 20, 30, 40, 50]
colors = [(255, 0, 0), (0, 255, 0), (0, 0, 255)]  # タプルのリスト

2. 要素の取得（インデックス）

Python
fruits = ["りんご", "バナナ", "みかん"]
print(fruits[0])   # りんご（最初の要素は0番目）
print(fruits[1])   # バナナ
print(fruits[-1])  # みかん（末尾から1番目）

インデックスは0から始まる

Pythonでは最初の要素は [0] です。[1] ではありません。存在しないインデックスを指定すると IndexError が発生します。

3. 要素の追加・長さ

Python
fruits = ["りんご", "バナナ"]
fruits.append("みかん")   # 末尾に追加
print(fruits)             # ['りんご', 'バナナ', 'みかん']
print(len(fruits))        # 3（要素の数）

4. スライス

Python
numbers = [10, 20, 30, 40, 50]
print(numbers[1:3])   # [20, 30]（1番目から3番目の手前まで）
print(numbers[:3])    # [10, 20, 30]（最初から3番目の手前まで）
print(numbers[2:])    # [30, 40, 50]（2番目から最後まで）

5. for文との組み合わせ

Python
fruits = ["りんご", "バナナ", "みかん"]
for fruit in fruits:

# バナナ

問2: 末尾から取得

問3: append と len

Python
data = [1, 2, 3]
data.append(99)

問4: スライス

Python
letters = ["a", "b", "c", "d", "e", "f"]
print(letters[1:4])
print(letters[:3])
print(letters[3:])

3つのスライスの結果はそれぞれ何ですか？

▶ 解答を見る

答え:

出力
['b', 'c', 'd']
['a', 'b', 'c']

# 星の画像を右に動かす（前回の「円が右に動く」と同じ構造）
import py5

img = None
x = 60
y = 120

def setup():
    global img
    py5.size(500, 240)
    img = py5.load_image("star.png")
    py5.image_mode(py5.CENTER)

def draw():
    global x
    py5.background(20, 30, 60)  # 毎フレーム背景で消す
    py5.image(img, x, y, 70, 70)
    x += 3  # 毎フレーム3pxずつ右へ
    if x > py5.width + 40:
        x = -40  # 画面右端まで来たら左から再登場

py5.run_sketch()

実行画面（時間の経過を1枚に重ねた図）

同じ星画像が x += 3 によって毎フレーム右に進んでいきます。下図はフレーム0/30/60/90/120を半透明で重ねた様子。

ポイント: なぜ background() を draw() の最初に呼ぶ？

draw() は毎フレーム呼ばれるので、何も消さないと前のフレームの画像が画面に残り続け「残像」になります。py5.background() を最初に呼ぶことで毎フレーム画面をリセットし、新しい位置にだけ画像が見えるようになります。

▲ 左: background() なし ── 星が動くたびに過去の姿が残る／右: background() あり ── 毎フレーム消してから描くので1つだけ見える

ポイント: load_image は setup()、image は draw()

画像の読み込みは重い処理なので setup() で1回だけ。draw() の中で毎フレーム load_image すると激重になります。読み込んだ img を global で共有して、draw() では描画のみ行います。

▲ 左: load_image を毎フレーム呼ぶ＝ディスクアクセスが秒60回 → カクカク／右: setup() で1回だけ読み込み、img をメモリに残して draw() から使う → ぬるぬる60FPS

ノート

py5で関数を使う: draw_house() で家を量産

家を描く処理を関数にまとめると、座標と色を変えるだけで何軒でも描けます。

Python ── draw_houses.py
# 関数で家を量産する
import py5

def draw_house(x, y, wall_color, roof_color):
    # 壁（四角形）
    py5.fill(*wall_color)
    py5.rect(x, y, 80, 60)
    # 屋根（三角形）
    py5.fill(*roof_color)
    py5.triangle(x - 10, y, x + 40, y - 40, x + 90, y)

def setup():
    py5.size(600, 300)
    py5.background(135, 206, 235)  # 空色
    draw_house(50, 200, (139, 69, 19), (255, 0, 0))
    draw_house(200, 200, (160, 82, 45), (0, 100, 0))
    draw_house(350, 200, (210, 180, 140), (0, 0, 200))

py5.run_sketch()

実行結果

空色の背景に、壁と屋根の色が違う家が3軒並びます。draw_house() を3回呼んでいるだけ ── 関数を1度書けば、引数を変えるだけで何軒でも描ける。

関数化のメリット

もし関数を使わなかったら、壁と屋根の描画コードを3回コピー＆ペーストする必要があります。関数化すると:

  • コードが短くなる（DRY原則: Don't Repeat Yourself）
  • 壁の幅を変えたいとき、1か所の修正で全ての家に反映される
  • 「この関数は何をするか」が名前から分かる

応用: リスト + 辞書 + 関数の組み合わせ

辞書のリストを関数で処理すると、データ管理が整理されます。次回（第5回）のCSVデータ処理はこのパターンの延長です。

Python ── combo_example.py
# リスト + 辞書 + 関数の組み合わせ
import py5

# 辞書のリスト（データ）
circles_data = [
    {"x": 100, "y": 150, "size": 60, "color": (255, 0, 0)},
    {"x": 250, "y": 150, "size": 80, "color": (0, 255, 0)},
    {"x": 400, "y": 150, "size": 50, "color": (0, 0, 255)},
]

def draw_circle(data):
    """辞書のデータから円を描画する"""
    py5.fill(*data["color"])
    py5.circle(data["x"], data["y"], data["size"])

def setup():
    py5.size(500, 300)
    py5.background(30)
    py5.no_stroke()
    for c in circles_data:
        draw_circle(c)

py5.run_sketch()

std4-3: 関数設計

課題内容

繰り返し使う描画処理を関数に分割したpy5プログラム std4_3.py を作成してください:

  1. 自分でオリジナルの描画関数を1つ以上定義する（例: draw_tree(x, y, color)、draw_star(x, y, size) など）
  2. 関数は引数を2つ以上受け取るようにする
  3. 関数を3回以上呼び出して、異なる引数で描画する
  4. for文やリストと組み合わせても良い

アイデア例
