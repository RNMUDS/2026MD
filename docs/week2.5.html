for 文の基本構造（5 秒で復習）

Python
for i in range(5):
    # i が 0,1,2,3,4 と順に変化しながらこのブロックを 5 回実行
    print(i)

覚えること 3 つだけ

  • range(N) は 0, 1, 2, ..., N-1 （N は含まない）
  • ブロックの中で i を「位置」「色」「サイズ」の式に使える
  • インデント (字下げ) されている行が 1 回ずつ実行される

問1

横一列に 5 つの円を並べる

基本問題

「等間隔に並んだ 5 つの円」

① 実行結果（コード連動アニメ）
コード行ハイライト ⇄ 円の出現が同期（7 秒ループ）
左: 実行中のコード行をオレンジ枠で表示／右: 対応する描画／下: 現在の変数値
② 考え方

変わるもの・変わらないもの

  • 変わるもの: 円の x 座標（60 → 130 → 200 → 270 → 340）
  • 変わらないもの: y 座標 (150)、半径 (25)

変わる x 座標を 60 + i*70 という式で表せれば、5 回繰り返すだけで済みます。

i60 + i*70x 座標
060 + 060
160 + 70130
260 + 140200
360 + 210270
460 + 280340
③ コード 
Python ── prob1.py
# prob1.py - 横一列に 5 つの円
import py5

def setup():
    py5.size(400, 300)
    for i in range(5):
        py5.ellipse(60 + i*70, 150, 50, 50)

py5.run_sketch()

1 行ずつ読む

  • size(400, 300) ── 400 × 300 のキャンバス
  • for i in range(5): ── i に 0,1,2,3,4 を順に入れて 5 回ループ
  • ellipse(60 + i*70, 150, 50, 50) ── x が 60 + i*70、y が 150 固定、幅 50・高さ 50 の円

問2

グラデーションの縦縞（色を i で計算する）

色 × ループ

「左から右へ黒→白に変化する 10 本の縦縞」

① 実行結果（コード連動アニメ）
10 反復のたびに「for → fill+rect」がハイライトされる（7 秒ループ）
左: コード行ハイライト／右: i ごとに縞が増える／下: 計算後の値
② 考え方

「位置」と「色」を同時に i で表現する

  • x 座標は i*40 （0, 40, 80, ..., 360）
  • 色（明るさ）は i*25 （0, 25, 50, ..., 225）
  • つまり 1 つのループ変数で 2 つのパラメータを制御している
ix = i*40fill = i*25イメージ
000 (真っ黒)
28050 (暗い灰)
5200125 (中間)
9360225 (明るい灰)
③ コード 
Python ── prob2.py
# prob2.py - グラデーションの縦縞
import py5

def setup():
    py5.size(400, 300)
    py5.no_stroke()
    for i in range(10):
        py5.fill(i*25)
        py5.rect(i*40, 0, 40, 300)

py5.run_sketch()

1 行ずつ読む

  • no_stroke() ── 縞の境界線を消す（グラデーションが滑らかに見える）
  • fill(i*25) ── 引数 1 つだとグレースケール (0=黒, 255=白)
  • rect(i*40, 0, 40, 300) ── 幅 40・高さ 300、x 座標は i*40

問3

階段状に下がる正方形（x も y も i で動かす）

2 軸を i で

「左上から右下へ 5 段の階段」

① 実行結果（コード連動アニメ）
x と y が同時に i*70 で進む様子（7 秒ループ）
左: コード行ハイライト／右: 斜めに 1 個ずつ追加／下: x, y 座標の値
② 考え方

x と y を同じ式で増やすと斜め

x 座標と y 座標が 同じペースで増えると、図形は右下方向への斜めに並びます。両方を 20 + i*70 で表現します。

③ コード 
Python ── prob3.py
# prob3.py - 階段状の正方形
import py5

def setup():
    py5.size(400, 400)
    for i in range(5):
        py5.rect(20 + i*70, 20 + i*70, 60, 60)

py5.run_sketch()

応用のヒント

x と y を 違う式にすると角度が変えられます。例えば rect(20 + i*70, 20 + i*30, ...) なら浅い斜め、rect(20 + i*30, 20 + i*70, ...) なら急な斜めになります。

問4

同心円（描画順がカギ）

サイズ × 描画順

「中心は同じで、サイズが減っていく 6 つの円」

① 実行結果（コード連動アニメ）
i が増える → d が減る → 内側に小さい円が積み上がる（7 秒ループ）
左: コード行ハイライト／右: 大→小の順で描かれる／下: 計算済み d の値
② 考え方

大きい円から先に描く ── これが超重要

py5 では 後から描いた図形が手前に重なります（順次制御）。「小さい円 → 大きい円」の順に描くと、大きい円が小さい円を覆い隠してしまいます。
正解は 「大きい円 → 小さい円」。 i が増えるごとに半径を減らす式を使います。

idiameter = 360 - i*60結果
0360一番大きな円（先に描かれる）
1300少し小さい
2240
3180
4120
560一番小さい円（最後に描かれる）
③ コード 
Python ── prob4.py
# prob4.py - 同心円
import py5

def setup():
    py5.size(400, 400)
    for i in range(6):
        d = 360 - i*60          # 直径: 360, 300, 240, 180, 120, 60
        py5.ellipse(200, 200, d, d)

py5.run_sketch()

失敗例: 半径が増えていく式にすると？

d = 60 + i*60 （0,60,120 → 60,120,180,...）にすると、最後に描くのが 一番大きい円。すべての小さい円が覆い隠されて、結果は一番大きな円 1 つだけに見えます。

同心円ごとに fill(i*40) を入れると、外側ほど黒、内側ほど白の同心円グラデーションになる。試してみよう。

問5

二重ループで格子模様

ネスト for

「5 × 5 の格子状に並ぶ 25 個の円」

① 実行結果（コード連動アニメ）
外側 x の中で内側 y が 5 回まわる ── これがネスト for（7 秒ループ）
左: コード行ハイライト／右: 列ごとに縦に埋まる、現在のセルはオレンジ枠／下: x と y の値
② 考え方

「2 つの軸」を 2 つの for で

1 次元（横一列）の発想を 2 軸に拡張するには、for x の中に for y を入れます。これをネスト（入れ子）と呼びます。

外側ループ × 内側ループの回数だけ、内側のブロックが実行されます。5 × 5 = 25 個。

③ コード 
Python ── prob5.py
# prob5.py - 二重ループで 5x5 格子
import py5

def setup():
    py5.size(400, 400)
    for x in range(5):
        for y in range(5):
            py5.ellipse(40 + x*80, 40 + y*80, 50, 50)

py5.run_sketch()

1 行ずつ読む

  • 外側 for x in range(5) ── 横方向 0〜4
  • 内側 for y in range(5) ── 縦方向 0〜4
  • 位置式: x 座標 = 40 + x*80, y 座標 = 40 + y*80
  • ループ全体は 5 × 5 = 25 回 実行される

🖐 やってみよう

2 つを同時に色に使うと、画面全体に色のマトリクスができます。 fill(x*50, y*50, 200) を ellipse の前に入れて実行してみよう。

Q1  for i in range(7): print(i) を実行すると、何が何回出力される？

0, 1, 2, 3, 4, 5, 6 の 7 回。range(N) は 0 から始まり N-1 までで、N は含まれない。

Q2  range(2, 6) をループするとどんな値が出る？

2, 3, 4, 5 の 4 つ。range(start, stop) は start を含み stop を含まない。

Q3  range(0, 10, 2) は何回ループする？

5 回（値は 0, 2, 4, 6, 8）。第 3 引数はステップ幅。

Q4  range(10, 0, -1) をループするとどんな順？

10, 9, 8, ..., 1 の逆順。0 は出力されない（stop は含まないルールは負のステップでも同じ）。

Q5  py5 で「縦に 4 つ円を並べる」コードの空欄を埋めよ:
for i in range( ___ ): py5.ellipse(200, 50 + i * ___, 30, 30)

4 と 50。x は 200 固定、y を 50, 100, 150, 200 と進めるので増分は 50。

Python
for i in range(4):
    py5.ellipse(200, 50 + i*50, 30, 30)
Q6  円の半径を 10, 20, 30, ... と増やしたい。r = ___ * (i + 1) の空欄は？

10。i = 0 のときに 10、i = 1 で 20、i = 2 で 30 にしたいので係数は 10。

もし (i + 1) ではなく i を使うなら、初項を 10 にずらすために r = 10 + i*10 と書く方法もあり。

Q7  for x in range(5): for y in range(3): の内側のブロックは何回実行される？

15 回（5 × 3 = 15）。外側が 1 回回るごとに内側は 3 回まわる。

Q8  ループ内で「偶数番目だけ円を塗る」にはどうする？

if i % 2 == 0: で偶数だけ fill を変える。

Python
for i in range(8):
    if i % 2 == 0:
        py5.fill(255, 200, 0)
    else:
        py5.fill(100)
    py5.ellipse(40 + i*45, 150, 35, 35)
Q9  for i in range(5): py5.rect(50 + i*70, 100, 50, 50) を 逆向き（右から左へ並ぶ）に変えるには？

x の式を 引き算に変えればよい。例:

Python
for i in range(5):
    py5.rect(330 - i*70, 100, 50, 50)

range(4, -1, -1) で i 自体を逆順にする方法もある。

Q10  二重ループで「市松模様（チェッカー）」を作りたい。空欄は？
if (x + y) % 2 == ___ : py5.fill(0)

0（または 1。0 にすれば「両座標の和が偶数のマス」が黒、奇数なら白）。

Python
for x in range(8):
    for y in range(8):
        if (x + y) % 2 == 0:
            py5.fill(0)
        else:
            py5.fill(255)
        py5.rect(x*50, y*50, 50, 50)
