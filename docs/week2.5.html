  • 変わらないもの: y 座標 (150)、半径 (25)

問4

同心円（描画順がカギ）

サイズ × 描画順

「中心は同じで、サイズが減っていく 6 つの円」

① 実行結果（コード連動アニメ）
0.00s / 7s
変数 i, 中間変数 d, ellipse 呼び出しを可視化（7 秒ループ）
下のパネル: i の値 → d の計算 → 実際の py5.ellipse 呼び出し
② 考え方

大きい円から先に描く ── これが超重要

py5 では 後から描いた図形が手前に重なります（順次制御）。「小さい円 → 大きい円」の順に描くと、大きい円が小さい円を覆い隠してしまいます。
正解は 「大きい円 → 小さい円」。 i が増えるごとに半径を減らす式を使います。

idiameter = 360 - i*60結果
0360一番大きな円（先に描かれる）
1300少し小さい
2240
3180
4120
560一番小さい円（最後に描かれる）
③ コード 
Python ── prob4.py
# prob4.py - 同心円
import py5

def setup():
    py5.size(400, 400)
    for i in range(6):
        d = 360 - i*60          # 直径: 360, 300, 240, 180, 120, 60
        py5.ellipse(200, 200, d, d)

py5.run_sketch()

失敗例: 半径が増えていく式にすると？

d = 60 + i*60 （0,60,120 → 60,120,180,...）にすると、最後に描くのが 一番大きい円。すべての小さい円が覆い隠されて、結果は一番大きな円 1 つだけに見えます。

🖐 やってみよう

同心円ごとに fill(i*40) を入れると、外側ほど黒、内側ほど白の同心円グラデーションになる。試してみよう。

問5

二重ループで格子模様

ネスト for

「5 × 5 の格子状に並ぶ 25 個の円」

① 実行結果（コード連動アニメ）
0.00s / 7s
外側 x と内側 y の値、各セルの ellipse 呼び出しを可視化（7 秒ループ）
下のパネル: 大きい x, y の値 → 25 通りの py5.ellipse(40+x*80, 40+y*80, 50, 50)
② 考え方

「2 つの軸」を 2 つの for で

1 次元（横一列）の発想を 2 軸に拡張するには、for x の中に for y を入れます。これをネスト（入れ子）と呼びます。

外側ループ × 内側ループの回数だけ、内側のブロックが実行されます。5 × 5 = 25 個。

③ コード 
Python ── prob5.py
# prob5.py - 二重ループで 5x5 格子
import py5

def setup():
    py5.size(400, 400)
    for x in range(5):
        for y in range(5):
            py5.ellipse(40 + x*80, 40 + y*80, 50, 50)

py5.run_sketch()

1 行ずつ読む

  • 外側 for x in range(5) ── 横方向 0〜4
  • 内側 for y in range(5) ── 縦方向 0〜4
  • 位置式: x 座標 = 40 + x*80, y 座標 = 40 + y*80
  • ループ全体は 5 × 5 = 25 回 実行される

🖐 やってみよう

2 つを同時に色に使うと、画面全体に色のマトリクスができます。 fill(x*50, y*50, 200) を ellipse の前に入れて実行してみよう。

かんたん 10 問（1 問 1 分）

Q1  for i in range(7): print(i) を実行すると、何が何回出力される？

0, 1, 2, 3, 4, 5, 6 の 7 回。range(N) は 0 から始まり N-1 までで、N は含まれない。

Q2  range(2, 6) をループするとどんな値が出る？

2, 3, 4, 5 の 4 つ。range(start, stop) は start を含み stop を含まない。

Q3  range(0, 10, 2) は何回ループする？

5 回（値は 0, 2, 4, 6, 8）。第 3 引数はステップ幅。

Q4  range(10, 0, -1) をループするとどんな順？

10, 9, 8, ..., 1 の逆順。0 は出力されない（stop は含まないルールは負のステップでも同じ）。

Q5  py5 で「縦に 4 つ円を並べる」コードの空欄を埋めよ:
for i in range( ___ ): py5.ellipse(200, 50 + i * ___, 30, 30)

4 と 50。x は 200 固定、y を 50, 100, 150, 200 と進めるので増分は 50。

Python
for i in range(4):
    py5.ellipse(200, 50 + i*50, 30, 30)
Q6  円の半径を 10, 20, 30, ... と増やしたい。r = ___ * (i + 1) の空欄は？

10。i = 0 のときに 10、i = 1 で 20、i = 2 で 30 にしたいので係数は 10。

もし (i + 1) ではなく i を使うなら、初項を 10 にずらすために r = 10 + i*10 と書く方法もあり。

Q7  for x in range(5): for y in range(3): の内側のブロックは何回実行される？

15 回（5 × 3 = 15）。外側が 1 回回るごとに内側は 3 回まわる。

Q8  ループ内で「偶数番目だけ円を塗る」にはどうする？

if i % 2 == 0: で偶数だけ fill を変える。

Python
for i in range(8):
    if i % 2 == 0:
        py5.fill(255, 200, 0)
    else:
        py5.fill(100)
    py5.ellipse(40 + i*45, 150, 35, 35)
Q9  for i in range(5): py5.rect(50 + i*70, 100, 50, 50) を 逆向き（右から左へ並ぶ）に変えるには？

x の式を 引き算に変えればよい。例:

Python
for i in range(5):
    py5.rect(330 - i*70, 100, 50, 50)

range(4, -1, -1) で i 自体を逆順にする方法もある。

Q10  二重ループで「市松模様（チェッカー）」を作りたい。空欄は？
if (x + y) % 2 == ___ : py5.fill(0)

0（または 1。0 にすれば「両座標の和が偶数のマス」が黒、奇数なら白）。

Python
for x in range(8):
    for y in range(8):
        if (x + y) % 2 == 0:
            py5.fill(0)
        else:
            py5.fill(255)
        py5.rect(x*50, y*50, 50, 50)

練習2

右に進むほど大きくなる円

キャンバス 400×300 に、横に 5 つ並ぶ円。右に行くほど大きくなる。x は 50, 125, 200, 275, 350、直径は 20, 40, 60, 80, 100。

x が 50→350、直径が 20→100 と同時に進む（6 秒ループ）

ヒント

x = 50 + i*75、直径 = 20 + i*20。1 つの i で 2 つのパラメータを動かす。

解答例を見る 
Python ── 解答例
import py5

def setup():
    py5.size(400, 300)
    for i in range(5):
        d = 20 + i*20
        py5.ellipse(50 + i*75, 150, d, d)

py5.run_sketch()

中間変数 d を使うと、横幅と縦幅を同じ値にしやすい。d なしで py5.ellipse(50+i*75, 150, 20+i*20, 20+i*20) でも OK。

練習3

赤と青が交互に並ぶ縞模様

キャンバス 400×300 に、幅 50 の長方形を 8 本並べる。偶数番目は赤、奇数番目は青。

i が偶数なら赤、奇数なら青（6 秒ループ）

ヒント

i % 2 == 0 で偶数判定。if/else で fill 色を切り替える。

解答例を見る 
Python ── 解答例
import py5

def setup():
    py5.size(400, 300)
    py5.no_stroke()
    for i in range(8):
        if i % 2 == 0:
            py5.fill(255, 0, 0)   # 赤
        else:
            py5.fill(0, 0, 255)   # 青
        py5.rect(i*50, 0, 50, 300)

py5.run_sketch()

% は剰余演算子。i % 2 は i を 2 で割った余り。偶数なら 0、奇数なら 1。

外から内へ縮む同心の正方形

キャンバス 400×400 に、5 つの正方形を 大きい順に重ねて描く。一番外側は 350×350、内側ほど一辺が 70 ずつ短くなる。

練習5

3 × 4 の格子に並ぶ 12 個の円

キャンバス 400×400 に、円を 3 列 × 4 行の格子状に並べる。x 座標は 75, 200, 325（3 列）、y 座標は 60, 160, 260, 360（4 行）。直径は 60。

外側 x の列が縦に埋まってから次の列へ（8 秒ループ）

ヒント

2 つの for をネストする（問5 と同じ）。x 方向 3 列・y 方向 4 行。
x = 75 + xx*125、y = 60 + yy*100。

解答例を見る 
Python ── 解答例
import py5

def setup():
    py5.size(400, 400)
    for xx in range(3):
        for yy in range(4):
            py5.ellipse(75 + xx*125, 60 + yy*100, 60, 60)

py5.run_sketch()

外側ループ xx = 0,1,2 (3 列) ／ 内側ループ yy = 0,1,2,3 (4 行) ／ 合計 3×4 = 12 個。
(Python の予約語と紛らわしいので変数名を xx, yy にしているが、i, j でも OK)

第 2.75 回 ── translate / push / pop を理解する

for 文で図形を並べるとき、座標の計算式が複雑になっていませんか？ translate() を使えば「原点を動かしてからローカル座標で描く」ことで、コードがぐっとシンプルになります。

  • translate(x, y) ── 原点を移動して相対座標で描画する基本
  • push() / pop() ── 座標系のセーブ＆ロードで translate の影響を限定する
  • for + push/pop で花・家・時計など複雑な図形を簡潔に配置する実践
  • rotate() との組み合わせで円周上に図形を並べるテクニック
