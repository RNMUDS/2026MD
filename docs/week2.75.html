translate / push / pop を理解する

py5 で「座標系を動かして描く」仕組みを学びます。translate() で原点を移動し、push() / pop() で座標系を保存・復元する技術は、複雑な図形を簡潔に書くための基本です。

0

このページの使い方

translate()、push()、pop() は座標変換の 3 大関数です。「図形の位置を計算で決める」のではなく「原点を動かしてからローカル座標で描く」という発想を身につけましょう。

  • ①完成図 ── 何を作るか（ゴール）
  • ②考え方 ── 図と表で「なぜ translate が使えるか」
  • ③コード ── 実装と 1 行ずつの解説

1

translate() ── 原点を動かす

基本概念

translate(x, y) とは？

translate(x, y) は、座標系の原点 (0, 0) を (x, y) へ移動させます。以降の描画命令はすべて新しい原点を基準にして動きます。

たとえ話：画用紙をずらす

机の上に画用紙がある。translate(100, 50) は「画用紙を右に 100、下に 50 ずらす」のと同じ。鉛筆を (0, 0) に置いても、画用紙上では (100, 50) に点が付く。

原点の移動イメージ 
Python
# translate なし ── rect の位置を直接計算
py5.rect(280, 160, 100, 80)

# translate あり ── 原点を (200,100) に動かしてからローカル座標で描く
py5.translate(200, 100)
py5.rect(80, 60, 100, 80)  # 画面上では (280, 160) に描かれる

translate は累積する！

translate(100, 0) を 2 回呼ぶと、原点は (200, 0) に移動します。元に戻したい場合は push() / pop() を使います（次のセクション）。

translate の累積を見る
0.00s / 8s
translate を呼ぶたびに原点がずれる（8 秒ループ）
2

push() / pop() ── 座標系の保存と復元

基本概念

push() と pop() のペア

たとえ話：セーブ & ロード

push() = 今の座標系をセーブする。pop() = セーブした状態にロード（復元）する。
push() と pop() の間でどんなに translate() しても、pop() した瞬間に元通り。

push/pop のイメージ 
Python
# push/pop の基本パターン
py5.push()               # 今の座標系をセーブ
py5.translate(150, 80)  # 原点を (150, 80) に移動
py5.ellipse(0, 0, 40, 40)  # (150,80) に円が描かれる
py5.pop()                # 座標系をロード → 原点は (0,0) に戻る

# この時点で原点は (0,0) なので、次の描画は影響を受けない
py5.rect(10, 10, 30, 30)  # (10,10) に四角が描かれる

push/pop なしだとどうなる？

translate(150, 80) の後に rect(10, 10, 30, 30) を描くと、rect は画面上の (160, 90) に描かれてしまいます。push/pop で囲めば、translate の影響を限定できます。

push/pop は「入れ子」にできる

push() の中でさらに push() を呼べます。それぞれの pop() で対応する push() の時点に戻ります。必ず push と pop は同じ数 にしてください。

問1

translate で横一列に花を 4 つ並べる

基本問題

「push/pop + translate で繰り返し配置」

① 完成図
4 つの花が等間隔で並ぶ
② 考え方

花のパーツは常に同じ ── 位置だけが違う

  • 花びら 4 枚 + 中心 = 全部で 5 つの円で 1 つの花
  • 花びらの座標は中心から見て (0,-25), (0,+25), (-25,0), (+25,0)
  • translate で花の中心を移動してから、同じ描画コードを使い回す
  • push/pop で囲めば、次の花に影響しない
i50 + i*100花の中心 x
050 + 050
150 + 100150
250 + 200250
350 + 300350
③ コード 
Python ── prob1.py
# prob1.py - push/pop + translate で花を 4 つ並べる
import py5

def setup():
    py5.size(400, 200)
    py5.background(204)

    for i in range(4):
        py5.push()                          # 座標系をセーブ
        py5.translate(50 + i*100, 100)   # 花の中心へ移動

        # 花びら (赤)
        py5.fill(255, 107, 107)
        py5.ellipse( 0, -25, 24, 24)   # 上
        py5.ellipse( 0,  25, 24, 24)   # 下
        py5.ellipse(-25,  0, 24, 24)   # 左
        py5.ellipse( 25,  0, 24, 24)   # 右

        # 中心 (黄)
        py5.fill(255, 184, 0)
        py5.ellipse(0, 0, 20, 20)

        py5.pop()                           # 座標系をロード → 原点が (0,0) に戻る

py5.run_sketch()

1 行ずつ読む

  • push() ── 現在の座標系を保存（この時点では原点 = (0,0)）
  • translate(50 + i*100, 100) ── 花の中心位置に原点を移動
  • ellipse(0, -25, ...) ── 「中心の上 25px」は原点からの相対位置
  • pop() ── 原点を (0,0) に戻す → 次のループの translate がリセットされる

やってみよう

push/pop を外してみよう。translate が累積するので、花がどんどん右にずれていくはず！

問2

入れ子の translate で家を描く

応用問題

「家のパーツを translate で相対配置」

① 完成図
translate で家の各パーツを相対配置
② 考え方

階層構造で考える

  • まず 家の基準点（左上 = (100, 180)）に translate する
  • 家の中で窓の位置に translate → 窓を「ローカル座標」で描く
  • 窓が終わったら pop → ドアの位置に translate → ドアを描く
  • 全部終わったら最外側の pop → 原点が (0,0) に戻る
③ コード 
Python ── prob2.py
# prob2.py - translate の入れ子で家を描く
import py5

def setup():
    py5.size(400, 400)
    py5.background(135, 206, 235)  # 空色

    # 地面
    py5.fill(34, 139, 34)
    py5.rect(0, 300, 400, 100)

    py5.push()                          # ★ 家の基準点をセーブ
    py5.translate(100, 180)            # 家の左上を原点にする

    # 家の本体 (幅200, 高さ150)
    py5.fill(222, 184, 135)
    py5.stroke(139, 69, 19)
    py5.rect(0, 0, 200, 150)

    # 屋根
    py5.fill(139, 0, 0)
    py5.triangle(-20, 0, 100, -100, 220, 0)

    # ── 左の窓 ──
    py5.push()
    py5.translate(20, 30)
    py5.fill(135, 206, 235)
    py5.stroke(51)
    py5.rect(0, 0, 40, 40)
    py5.line(20, 0, 20, 40)          # 十字の縦線
    py5.line(0, 20, 40, 20)          # 十字の横線
    py5.pop()

    # ── 右の窓 ──
    py5.push()
    py5.translate(140, 30)
    py5.fill(135, 206, 235)
    py5.stroke(51)
    py5.rect(0, 0, 40, 40)
    py5.line(20, 0, 20, 40)
    py5.line(0, 20, 40, 20)
    py5.pop()

    # ── ドア ──
    py5.push()
    py5.translate(70, 80)
    py5.fill(139, 69, 19)
    py5.rect(0, 0, 60, 70)
    py5.fill(255, 184, 0)
    py5.ellipse(50, 38, 8, 8)       # ドアノブ
    py5.pop()

    py5.pop()                           # ★ 家の基準点をロード

py5.run_sketch()

ポイント

  • 外側の push/pop が「家全体」の座標系
  • 内側の push/pop が「窓」「ドア」のローカル座標系
  • 窓の描画コードは左も右もまったく同じ ── translate の位置だけが違う

やってみよう

外側の translate(100, 180) を translate(50, 100) に変えてみよう。家全体が左上に移動するはず。パーツ間の位置関係は崩れない！

問3

for + push/pop で時計の目盛りを描く

応用問題

「translate + rotate で円周上に配置」

① 完成図
12 本の目盛りを translate + rotate で描く
② 考え方

rotate = 座標系を回転させる

  • translate(200, 200) で原点をキャンバス中心に移動
  • 12 本の目盛りは 30 度 (= 360/12) ずつ回転した同じ線
  • push() → rotate(i * 30度) → 線を描く → pop()
  • ラジアン変換: py5.radians(角度)
ii * 30角度 (度)位置
00012 時
130301 時
260602 時
390903 時
... 30度ずつ増えて 12 本
③ コード 
Python ── prob3.py
# prob3.py - translate + rotate で時計の目盛り
import py5

def setup():
    py5.size(400, 400)
    py5.background(204)
    py5.stroke_weight(3)

    # 文字盤
    py5.fill(240)
    py5.ellipse(200, 200, 340, 340)

    py5.push()                          # ★ 外側の push
    py5.translate(200, 200)            # 原点をキャンバス中心に

    for i in range(12):
        py5.push()                      # ★ 各目盛り用の push
        py5.rotate(py5.radians(i * 30))  # i*30 度回転
        py5.line(0, -160, 0, -130)    # 上方向に線を引く
        py5.pop()                       # ★ 回転をリセット

    # 中心の丸
    py5.fill(51)
    py5.ellipse(0, 0, 10, 10)

    py5.pop()                           # ★ 外側の pop

py5.run_sketch()

1 行ずつ読む

  • translate(200, 200) ── 原点をキャンバス中心に → 回転の軸もここ
  • rotate(py5.radians(i * 30)) ── 座標系全体を i*30 度回転
  • line(0, -160, 0, -130) ── 「上方向に短い線」を引く。回転により位置が変わる
  • push/pop のおかげで、各ループの rotate が次のループに影響しない

やってみよう

range(12) を range(60)、角度を i * 6 に変えると秒の目盛り（60 本）になる。太さを stroke_weight(1) にすると時計らしくなる。

Q

理解度チェック ── 小問 8 題

各問、答えを考えてから開いてください。

Q1 translate(100, 50) の後に rect(0, 0, 30, 30) を描くと、画面上の位置は？

(100, 50) に四角が描かれる。
translate は原点を移動するので、rect(0,0,...) は新しい原点 = 画面上の (100,50) に描かれる。

Q2 translate(100, 0) を 3 回呼んだ後、原点はどこ？

(300, 0)。translate は累積する。100 + 100 + 100 = 300。

Q3 push/pop を使わずに translate の効果を元に戻すには？

translate(-100, -50) のように逆方向に translate する。ただし push/pop を使うほうが安全で可読性も高い。

Q4 push() は translate 以外にも何を保存する？

fill, stroke, strokeWeight, rotate, scale など、すべての描画設定を保存する。pop() ですべて元に戻る。色を変えたい箇所だけ push/pop で囲むテクニックも有効。

Q5 次のコードで 2 つ目の円はどこに描かれる？
translate(50,50); ellipse(0,0,20,20); translate(100,0); ellipse(0,0,20,20);

1 つ目: (50, 50)。2 つ目: (150, 50)。
2 回目の translate は 1 回目に累積するので、原点は (50+100, 50+0) = (150, 50) になる。

Q6 push() を呼んだのに pop() を呼び忘れるとどうなる？

py5 が内部スタックにゴミが溜まる。動作はおかしくならない場合もあるが、保存した状態が解放されず、多く繰り返すとエラーになる可能性がある。必ずペアにすること。

Q7 for ループの中で push/pop を使う理由は？

ループのたびに translate/rotate が累積してしまうのを防ぐため。push/pop で囲めば、各反復が「まっさらな座標系」から始められる。

Q8 translate(200,200) してから rotate(py5.radians(45)) すると、回転の中心はどこ？

(200, 200)。rotate は常に現在の原点を中心に回転する。先に translate で原点を移動しているので、画面上の (200,200) が回転の中心になる。

練習問題

練習1

信号機を描く

キャンバス 300×400 に縦型の信号機を描く。本体は黒い角丸四角、中に赤・黄・緑の 3 つの円を translate で配置する。

translate で赤・黄・緑をローカル座標で配置

ヒント

本体の左上に translate してから、各ライトの中心位置に push/translate/pop で配置する。3 つのライトの描画コードはほぼ同じ（色だけ違う）。

解答例を見る 
Python ── 解答例
import py5

def setup():
    py5.size(300, 400)
    py5.background(204)

    # 支柱
    py5.fill(51)
    py5.no_stroke()
    py5.rect(135, 300, 30, 100)

    py5.push()
    py5.translate(90, 40)             # 本体の左上

    # 本体
    py5.fill(34)
    py5.rect(0, 0, 120, 280, 20)

    # ライトの色と y 位置
    colors = [(255,51,51), (255,204,0), (51,204,51)]
    for i in range(3):
        py5.push()
        py5.translate(60, 70 + i*80)  # 各ライトの中心
        py5.fill(colors[i][0], colors[i][1], colors[i][2])
        py5.ellipse(0, 0, 70, 70)
        py5.pop()

    py5.pop()

py5.run_sketch()
練習2

push/pop + for で 2 列 × 3 行の街を描く

キャンバス 600×500 に、問 2 の家を 2 列 × 3 行に並べる。各家は translate で配置し、描画コードは 1 つだけにする。

translate + for のネストで同じ家を 6 軒配置

ヒント

二重ループ for col in range(3): for row in range(2): で、translate(col*180 + 50, row*150 + 50) して家の描画関数を呼ぶ。家の描画は「ローカル座標で書く」のがポイント。

解答例を見る 
Python ── 解答例
import py5

def draw_house():
    # 本体 (幅120, 高さ90)
    py5.fill(222, 184, 135)
    py5.stroke(139, 69, 19)
    py5.rect(0, 0, 120, 90)
    # 屋根
    py5.fill(139, 0, 0)
    py5.triangle(-10, 0, 60, -50, 130, 0)
    # ドア
    py5.fill(139, 69, 19)
    py5.rect(45, 45, 30, 45)

def setup():
    py5.size(600, 500)
    py5.background(135, 206, 235)
    py5.fill(34, 139, 34)
    py5.no_stroke()
    py5.rect(0, 400, 600, 100)

    for col in range(3):
        for row in range(2):
            py5.push()
            py5.translate(col*180 + 50, row*150 + 50)
            draw_house()
            py5.pop()

py5.run_sketch()

draw_house() は常にローカル座標 (0,0) 基準で描く。translate で家の位置を変えるだけで、何軒でも配置できる。これが translate + push/pop の真価。

練習3

translate + rotate で太陽系を描く

キャンバス 500×500 の中心に太陽を描き、3 つの惑星を異なる角度で円軌道上に配置する。軌道は no_fill() の円で描く。

translate(中心) → rotate(角度) → translate(軌道半径, 0) で惑星を配置

ヒント

各惑星: push() → rotate(角度) → translate(軌道半径, 0) → ellipse(0,0,...) → pop()。rotate + translate の組み合わせで円周上の任意の点に配置できる。

解答例を見る 
Python ── 解答例
import py5

def setup():
    py5.size(500, 500)
    py5.background(17)

    py5.push()
    py5.translate(250, 250)          # 原点を中心に

    # 軌道を描画
    py5.no_fill()
    py5.stroke(51)
    for r in [80, 140, 200]:
        py5.ellipse(0, 0, r*2, r*2)

    # 太陽
    py5.no_stroke()
    py5.fill(255, 184, 0)
    py5.ellipse(0, 0, 50, 50)

    # 惑星データ: (軌道半径, 角度, サイズ, R, G, B)
    planets = [
        (80,  45,  20, 79, 195, 247),
        (140, 160, 28, 255, 107, 107),
        (200, 280, 16, 118, 185, 0),
    ]
    for dist, angle, sz, r, g, b in planets:
        py5.push()
        py5.rotate(py5.radians(angle))
        py5.translate(dist, 0)
        py5.fill(r, g, b)
        py5.ellipse(0, 0, sz, sz)
        py5.pop()

    py5.pop()

py5.run_sketch()

rotate で方向を決め、translate で距離を進む。この「回してから進む」パターンは円周上の配置で非常に強力。
