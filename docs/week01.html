ターミナルとは

ターミナルとは、PCに命令を出すための特別な画面です。普段使うマウスでクリックする操作とは違い、文字を打ち込んでPCを操作します。キーボードで命令文（コマンド）を入力すると、PCがその通りに動きます。

普段のPC操作は「メニューから料理を選ぶ」ようなもの。ターミナルは「シェフに直接注文する」ようなもの。自由度が高い分、正確に伝える必要があります。

CS基礎

ファイルシステムと木構造

コンピュータの中のファイルは木構造（ツリー構造）で整理されています。これはコンピュータサイエンスの最も基本的なデータ構造の1つです。

本棚に例えると：本棚（ルートディレクトリ）の中に棚（ディレクトリ）があり、棚の中に本（ファイル）がある。棚の中にさらに小さな仕切り（サブディレクトリ）を作ることもできる。 
木構造の例
/ （ルート = 木の根）
├── Users/
│   └── username/        ← ホームディレクトリ（~）
│       ├── Documents/
│       ├── Downloads/
│       └── Desktop/
├── Applications/
└── System/

なぜ木構造が重要か

  • ファイルシステム（今学んでいるもの）── ディレクトリとファイルの階層
  • Webページ（HTML）── タグの入れ子構造も木構造
  • AI・機械学習── 決定木、探索木など至るところに登場
  • 木構造を理解するとパス（/Users/username/Documents/...）の意味が腑に落ちる

- ディレクトリとは ＿＿＿＿＿＿＿＿＿＿＿＿＿＿ のこと（Windowsでは＿＿＿＿と呼ぶ）
- 「/」（スラッシュ）は ＿＿＿＿＿＿ を意味する
- 「~」（チルダ）は ＿＿＿＿ディレクトリを意味する
- /Users/rn/Documents/Lecture というパスは、ルートから順に ＿＿ → ＿＿ → ＿＿＿＿＿＿ → ＿＿＿＿＿ と辿った場所を表す

もっと知りたい人へ: Macのカーネル「XNU」とは

現在のMacのカーネルは XNU です。

XNU の位置づけ

macOS の内部構造は以下のようになっています。

  • Darwin ── macOS の土台（オープンソース部分）
  • XNU ── Darwin の中核カーネル（OSの心臓）

XNU = ハイブリッドカーネル

XNU は2つの技術を合体させた設計です。

構成要素役割
Machマイクロカーネル的な仕組み（プロセス管理、メモリ管理）
BSDUnix系の機能（ファイルシステム、ネットワーク、セキュリティ）

名前の由来: XNU = "X is Not Unix"（再帰的な命名）

Mac の全体構造

アプリ（Safari, Chrome など）
      ↓
macOS の API（Cocoa など）
      ↓
Darwin（OS の土台）
      ↓
XNU カーネル ← ★ OS の心臓
      ↓
ハードウェア（CPU, メモリ）

Linux のカーネルは「モノリシックカーネル」、Windows は「NT カーネル」── それぞれ設計思想が異なります。興味があれば調べてみましょう。

全体の流れ： Homebrew でツール（uv, Java, VSCode）を導入 → uv で Python をインストール → uv venv で仮想環境を作る → その中に uv pip install でライブラリ（py5）を追加 → プログラムを書いて実行

パッケージはどこから来る？ ── ネットワークの基礎

uv pip install py5 を実行すると、一瞬でライブラリがインストールされます。裏で何が起きているのでしょうか？

あなたのPC
（クライアント） → DNS
名前→IPアドレス → HTTPS通信
暗号化された経路 → PyPI サーバ
pypi.org → ダウンロード
.venv に保存
用語意味身近な例え
クライアント サービスを要求する側（あなたのPC） レストランの客
サーバ サービスを提供する側（PyPI、Webサイトなど） レストランの厨房
IPアドレス インターネット上のコンピュータの住所（例: 151.101.128.223） 建物の住所
ポート 1つのIPアドレス上の窓口番号（例: 443 = HTTPS） 建物の中の部屋番号
URL リソースの場所を示す文字列（例: https://pypi.org/project/py5/） 「東京都○○区△△ビル3階」のような完全な住所
DNS ドメイン名（pypi.org）をIPアドレスに変換する仕組み 電話帳（名前→電話番号）
HTTP Webの通信ルール（プロトコル）。データのやり取りの決まり事 郵便の書式ルール
HTTPS HTTPに暗号化を加えたもの。盗聴を防ぐ 中身が見えない封筒で郵送

リクエストとレスポンス

すべてのネットワーク通信は「要求（リクエスト）」と「応答（レスポンス）」の繰り返しです。

  • uv pip install py5 → PCが PyPI サーバに「py5をください」とリクエスト
  • PyPI サーバが py5 のファイルをレスポンスとして返す
  • uv がファイルを受け取り、.venv の中にインストール

ブラウザでWebサイトを見るのも、Slackでメッセージを送るのも、すべて同じリクエスト・レスポンスの仕組みです。

- サービスを要求する側を ＿＿＿＿＿＿ 、提供する側を ＿＿＿ と呼ぶ
- uv pip install py5 を実行すると、PCは ＿＿＿＿ というサーバに接続してパッケージを取得する
- IPアドレスとは ＿＿＿＿＿＿＿＿＿＿ のこと。例: 151.101.128.223
- DNSの役割は ＿＿＿＿＿＿（例: pypi.org）を ＿＿＿＿＿＿＿ に変換すること
- HTTPSの「S」は ＿＿＿＿ の意味で、通信内容が ＿＿＿＿ される
- ネットワーク通信はすべて ＿＿＿＿＿（要求）と ＿＿＿＿＿（応答）の繰り返しである

import py5                # py5ライブラリを読み込む

def setup():              # プログラム開始時に1回だけ実行される関数
    py5.size(400, 300)    # ウィンドウサイズ: 幅400px、高さ300px

    py5.ellipse(300, 100, 100, 100)  # 円: 中心(300,100)、幅100、高さ100

py5.run_sketch()           # プログラムを実行する
関数意味引数
py5.size(w, h)ウィンドウサイズを設定w=幅、h=高さ（ピクセル）
py5.background(r, g, b)背景色を設定RGB値（各0〜255）
py5.fill(r, g, b)塗りつぶし色を設定以降の図形に適用される
py5.rect(x, y, w, h)矩形を描画左上座標(x,y)、幅w、高さh
py5.ellipse(x, y, w, h)楕円を描画（w=hで円）中心座標(x,y)、幅w、高さh
py5.triangle(x1,y1, x2,y2, x3,y3)三角形を描画3つの頂点座標
py5.line(x1,y1, x2,y2)線を描画始点と終点
py5.stroke(r, g, b)輪郭線の色RGB値
py5.no_stroke()輪郭線を消すなし
py5.no_fill()塗りつぶしを消すなし

ノート CS基礎 穴埋め解答

ファイルシステムと木構造:

  • コンピュータのファイルは 木構造（ツリー構造）で整理されている
  • ディレクトリとは ファイルやデータを整理して保存する場所 のこと（Windowsではフォルダと呼ぶ）
  • 「/」は 階層の区切り を意味する
  • 「~」は ホームディレクトリを意味する
  • 木構造はファイルシステム以外にも Webページ（HTML） や AI・機械学習（決定木） で使われている

ネットワークの仕組み:

  • サービスを要求する側を クライアント、提供する側を サーバ と呼ぶ
  • uv pip install py5 を実行すると、PCは PyPI というサーバに接続する
  • IPアドレスとは インターネット上のコンピュータの住所 のこと
  • DNSの役割は ドメイン名 を IPアドレス に変換すること
  • HTTPSの「S」は Secure の意味で、通信内容が 暗号化 される
  • ネットワーク通信はすべて リクエスト（要求）と レスポンス（応答）の繰り返し

google.comにアクセスしたとき:

  1. ブラウザがDNSに問い合わせて google.com のIPアドレスを取得する
  2. そのIPアドレスにHTTPSリクエストを送る
  3. Googleのサーバがレスポンス（HTMLファイル）を返し、ブラウザが画面に表示する
ノート アウトカム穴埋め解答
  • ものづくりにおいて今後重要なキーワード: アウトカム
  • そのキーワードの意味: 作ったものによって誰かの状態・行動が変わること。アプリを作っただけ（アウトプット）ではなく、それを使った人に変化が起きて初めて価値が証明される。
ノート Slackマナー穴埋め解答
  • メンションやDMを受けたら 24時間以内に返信する
  • すぐに回答できないときは「確認しました、後で返信します」と一言返す
  • メッセージを読んだら リアクション（絵文字） をつけて「読んだ」と伝える
  • 返信はチャンネルに直接書かず スレッド返信 で行う。理由はチャンネルのタイムラインが分断されず話題ごとにまとまるため

予備知識

macOSとは

AppleのPC「Mac」で使われているOS（オペレーティングシステム）。UNIXから派生したOSとしてターミナル操作やオープンソースソフトウェアのサポートが可能。

データサイエンティスト・AI/ソフトウェアエンジニアがMacを使う理由

  • UNIXベースによるLinuxとの高い親和性
  • ターミナルの操作性
  • PythonやRなどの言語との相性
  • Homebrewによるパッケージ管理の簡便性
  • DockerやGitなどの開発ツールの互換性
  • 高品質なハードウェア設計
  • 安定した動作とセキュリティの高さ
  • Unixと同様のコマンドを使用可能

補足

Windowsにも優れた点（GPU環境、ゲーム開発、企業内システム対応など）はあるが、MacはとくにUNIX的環境とGUIの共存が強力であり、データサイエンス・AI・ソフトウェア開発の現場で好まれる。
