AD/DA変換 ─ アナログとデジタルの橋渡し

音は空気の振動（アナログ信号）です。コンピュータで扱うにはデジタル信号に変換する必要があります。

  • AD変換（Analog-to-Digital）: マイクで拾った音（連続的な波）を、一定間隔で測定し数値化する
  • 標本化（サンプリング）: 波の高さを1秒間に何回測定するか → サンプリングレート（CDは44,100Hz = 1秒に44,100回）
  • 量子化: 測定した値を離散的な数値に丸める → ビット深度（CDは16bit = 65,536段階）
  • DA変換（Digital-to-Analog）: デジタルデータをスピーカーで再びアナログの振動に変換して音を鳴らす
音のデジタル化は「パラパラ漫画」と同じです。連続的な動きを1コマずつ描いて記録し（AD変換）、高速でめくると動いて見える（DA変換）。コマ数が多いほど滑らかになります。
AD変換アナログ信号をデジタル数値に変換 サンプリングレート1秒間の測定回数 (CD: 44,100Hz) ビット深度音量の段階数 (CD: 16bit = 65,536段階) FFT高速フーリエ変換: 波形を周波数成分に分解 周波数 (Hz)1秒間の振動回数。高いほど高い音

pydubの基本 ─ AudioSegment

ノート

pydubとは

pydub は音声ファイルをPythonで簡単に扱うためのライブラリです。MP3、WAV、OGGなど様々な形式に対応しています。音声データは AudioSegment オブジェクトとして扱います。

まず、適当なMP3ファイルを ~/MD1/no6/ に配置してください。フリー音源サイトからダウンロードしたものでOKです。

Python ── MP3再生
from pydub import AudioSegment
from pydub.playback import play

# MP3ファイルを読み込む
audio = AudioSegment.from_file("sample.mp3")

# 音声情報を表示
print(f"長さ: {len(audio)}ミリ秒")
print(f"チャンネル数: {audio.channels}")
print(f"サンプリングレート: {audio.frame_rate}Hz")
print(f"ビット深度: {audio.sample_width * 8}bit")

# 再生
play(audio)

AudioSegmentの基本操作

  • len(audio) → 長さ（ミリ秒）
  • audio[:5000] → 最初の5秒を切り出し
  • audio + audio2 → 2つの音声を連結
  • audio.apply_gain(-10) → 音量を10dB下げる
  • audio.overlay(audio2) → 2つの音声を重ねる
  • audio.export("output.wav", format="wav") → ファイルに保存

Step 2: 周波数を指定して音を作る（約15分）

正弦波で音を生成する

ノート

音の三要素

  • 高さ（ピッチ）: 周波数（Hz）で決まる。440Hz = ラの音
  • 大きさ（音量）: 振幅で決まる。波の高さが大きいほど音が大きい
  • 音色（ティンバー）: 波の形で決まる。正弦波 = 純音、矩形波 = ブザー音
Python ── 正弦波の生成
import numpy as np
from pydub import AudioSegment
from pydub.playback import play

def create_sine_wave(frequency, duration_ms, sample_rate=44100):
    """指定した周波数・長さの正弦波を生成する"""
    num_samples = int(sample_rate * duration_ms / 1000)
    t = np.linspace(0, duration_ms / 1000, num_samples)
    wave = np.sin(2 * np.pi * frequency * t)
    wave = (wave * 32767).astype(np.int16)
    audio = AudioSegment(wave.tobytes(),
                         frame_rate=sample_rate,
                         sample_width=2,
                         channels=1)
    return audio

# ラの音（440Hz）を1秒鳴らす
la_sound = create_sine_wave(440, 1000)
play(la_sound)

Python ── simpleaudioでの再生
import simpleaudio as sa
import numpy as np

def play_tone(frequency, duration_ms=300, sample_rate=44100):
    """ノンブロッキングで音を再生する"""
    num_samples = int(sample_rate * duration_ms / 1000)
    t = np.linspace(0, duration_ms / 1000, num_samples, False)
    wave = np.sin(2 * np.pi * frequency * t)
    audio = (wave * 32767).astype(np.int16)
    play_obj = sa.play_buffer(audio, 1, 2, sample_rate)
    return play_obj  # ノンブロッキング: 再生中でも次の処理へ

Step 4: FFTで音を可視化する（約25分）

FFT（高速フーリエ変換）入門

ノート

FFTとは何か

FFT（Fast Fourier Transform / 高速フーリエ変換）は、音の波形データを周波数成分に分解する数学的手法です。

例えば、ギターの「ラ」の音は440Hzだけでなく、880Hz、1320Hz…など複数の周波数が混ざっています。FFTを使うと「この音にはどんな周波数がどれだけ含まれているか」がわかります。

虹と同じです。白い光（音の波形）をプリズム（FFT）に通すと、赤・橙・黄…と色（周波数）に分解されます。

ステップバイステップ: FFTの手順

  1. 音声データを読み込む → サンプル値の配列（波形）を得る
  2. 窓（ウィンドウ）を切り出す → 全体ではなく一部分（4096サンプルなど）を取り出す
  3. np.fft.fft() を実行 → 複素数の配列が返る
  4. np.abs() で振幅を取る → 各周波数の強さ（マグニチュード）
  5. 前半だけ使う → FFT結果は左右対称なので前半のみ有効

FFTスペクトルをpy5で描画する

ノート 
Python ── FFT可視化
import numpy as np
import py5
from pydub import AudioSegment

# 音声ファイルを読み込み、モノラルに変換
audio = AudioSegment.from_file("sample.mp3").set_channels(1)
samples = np.array(audio.get_array_of_samples())

# FFTパラメータ
WINDOW = 4096
fft = np.fft.fft(samples[:WINDOW])
magnitude = np.abs(fft[:WINDOW // 2])

def setup():
    py5.size(800, 400)
    py5.color_mode(py5.HSB, 255)
    py5.no_loop()

def draw():
    py5.background(0)

    # スペクトルを棒グラフとして描画
    for i in range(0, len(magnitude), 8):
        x = py5.remap(i, 0, len(magnitude), 0, py5.width)
        y = py5.remap(magnitude[i], 0, magnitude.max(), py5.height, 0)
        # 色を周波数に応じて変化させる（HSBカラー）
        py5.stroke(py5.remap(i, 0, len(magnitude), 0, 255), 200, 255)
        py5.line(x, py5.height, x, y)

py5.run_sketch()

コードの読み方

  • set_channels(1) → ステレオをモノラルに変換（解析を簡単にするため）
  • WINDOW = 4096 → 一度に解析するサンプル数。2のべき乗にするとFFTが高速
  • fft[:WINDOW // 2] → FFT結果の前半だけ取る（後半は対称なので不要）
  • remap() → 値の範囲を別の範囲に変換する（py5の便利関数）

なぜドミソは美しく響くのか？

ノート

和音と周波数比

和音が「美しい」と感じるかどうかは、周波数の比率で決まります。比率がシンプルな整数比に近いほど「協和」して美しく聞こえます。

和音音名周波数 (Hz)ド基準の比率近似整数比
ドC4261.631.001
ミE4329.631.265/4
ソG4392.001.503/2

ミ/ド = 5/4、ソ/ド = 3/2 → シンプルな整数比 = 美しい和音（協和音）

Python ── ドミソ和音の生成
from pydub.generators import Sine
from pydub.playback import play

# ドミソの3つの音を生成
do = Sine(261.63).to_audio_segment(duration=1000)
mi = Sine(329.63).to_audio_segment(duration=1000)
so = Sine(392.00).to_audio_segment(duration=1000)

# 音量を下げてから重ねる（クリッピング防止）
chord = do.apply_gain(-10).overlay(mi.apply_gain(-10)).overlay(so.apply_gain(-10))
play(chord)

# 周波数比を確認
print(f"ミ÷ド = {329.63 / 261.63:.3f} ≈ 5/4 = {5/4:.3f}")
print(f"ソ÷ド = {392.00 / 261.63:.3f} ≈ 3/2 = {3/2:.3f}")

実験してみよう: 不協和音

周波数比が複雑な音を重ねると「不協和音」になります。

Python ── 不協和音の実験
# ド + ド# → 半音の重なり = 不協和音
do = Sine(261.63).to_audio_segment(duration=1000)
do_sharp = Sine(277.18).to_audio_segment(duration=1000)
dissonance = do.apply_gain(-10).overlay(do_sharp.apply_gain(-10))
play(dissonance)
print(f"ド#÷ド = {277.18 / 261.63:.3f} → 複雑な比率 = 不協和")
縄跳びの二重回しと同じです。2人が同じリズム（3:2など）で跳ぶと美しく見える。バラバラのリズムだと混乱する。音の波も同じで、山と山が規則的に重なると心地よく聞こえます。

from pydub import AudioSegment

audio = AudioSegment.from_file("sample.mp3").set_channels(1)
samples = np.array(audio.get_array_of_samples())

WINDOW = 4096
HOP = 2048  # ウィンドウをずらす幅

    mag = np.abs(fft[:WINDOW // 2])
    spectrums.append(mag)

CS/DS 基礎知識

AD/DA変換（アナログ-デジタル変換）

すべてのメディアはデジタル化される

コンピュータは0と1しか扱えません。音・画像・テキストなど、あらゆる情報を数値（デジタル）に変換する必要があります。

メディアデジタル化の方法単位
音標本化（サンプリング）+ 量子化44,100 Hz / 16bit
画像ピクセル分割 + RGB値1920x1080 / 24bit
テキスト文字コード割り当てASCII / UTF-8
温度計で気温を記録するのと同じです。アナログの温度計は連続的に変化しますが、1時間ごとに値を読み取って「9時:20℃、10時:22℃…」とメモすれば、それがAD変換です。

文字コード ─ すべてはデジタル

文字も数値で表現する

  • ASCII: 英数字・記号を7bit（128文字）で表現。'A' = 65、'a' = 97、'0' = 48
  • UTF-8: 世界中の文字を可変長（1〜4バイト）で表現。日本語は3バイト/文字
  • 絵文字も数値: 「😊」は U+1F60A = 128522

FFT（高速フーリエ変換）

時間領域 → 周波数領域の変換

フーリエ変換は「どんな複雑な波も、単純な正弦波の組み合わせで表せる」という数学的定理に基づいています。

  • 時間領域: 横軸=時間、縦軸=振幅 → 「波形」
  • 周波数領域: 横軸=周波数、縦軸=強さ → 「スペクトル」
  • DFT（離散フーリエ変換）: デジタル信号に適用可能なフーリエ変換。計算量 O(N²)
  • FFT: DFTを高速化したアルゴリズム。計算量 O(N log N) → 実用的な速度
カクテルの味から「レモン30%、砂糖20%、ラム50%」と材料を特定するようなものです。混ざった音（カクテル）を個々の周波数（材料）に分解します。

フィルタリング概念

不要な周波数を除去する

FFTで周波数成分に分解した後、不要な成分を除去して元に戻す（逆FFT）ことで音を加工できます。

  • ローパスフィルタ: 低い周波数だけ通す → こもった音、ノイズ除去
  • ハイパスフィルタ: 高い周波数だけ通す → シャリシャリした音
  • バンドパスフィルタ: 特定の範囲だけ通す → 電話の音声（300Hz〜3400Hz）
コーヒーフィルターと同じです。豆の粒子（不要な周波数）を取り除いて、液体（欲しい周波数）だけを通します。

サンプリング定理（ナイキストの定理）

なぜサンプリングレートは44,100Hz？

ナイキストの定理: 周波数 f の音を正しくデジタル化するには、2f 以上のサンプリングレートが必要です。

  • 人間の可聴域: 約20Hz〜20,000Hz
  • 20,000Hz × 2 = 40,000Hz → 少し余裕を持って 44,100Hz
  • サンプリングレートが不足すると「エイリアシング」（偽の周波数が現れる）が発生
