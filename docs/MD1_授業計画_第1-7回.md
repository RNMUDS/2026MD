# MD1 授業計画（第1〜7回）

## 科目概要

**Media creation Data design 1（MD1）**

- 1年次・1学期・必修科目
- 担当：中村亮太
- 月曜3・4時限（13:10〜16:40 / 計200分）
- 計画対象時間：各回 **120分**（残り80分は休憩・バッファ・質問対応・環境トラブル等）

---

## 授業方針

> AIの使用は常に許可する。問われるのは「作れるか？」ではなく「説明できるか？」

1. **制作物そのものは評価の中心にしない**
評価の中心は、説明・設計・入力と期待結果・修正理由である

2. **AIを使ったことは問題ではない。説明できないことが問題である**
AIは答え生成機ではなく、読解・比較・修正の対象を増やす教材生成機として使う

3. **実践から入る**
まず動かす → 「なぜこうなる？」を考える → 知識として整理する

4. **Playful Learning**
遊びながら学ぶ。グループで教え合うことが最も効果的な学習

5. **毎回の提出物（3点セット）**
    - コード
    - 考え方メモ（なぜこう書いたか）
    - 入力と期待結果（何を試し、何が起こるはずか）

|---|------|-------------|
| G1 | 開発環境を構築し、プログラムを実行できる | 基礎力 |

## 毎回の授業フレーム（120分）

| Phase | 時間 | 内容 |
|-------|------|------|
| **Play** | 30分 | 完成コードを動かす。「何が起きた？」をグループで話す |
| **Explore** | 30分 | コードを変える・AIの出力と比べる。「変えたらどうなる？」 |
| **Trace** | 25分 | トレース表を書く。1行ずつ追う |
| **Explain** | 20分 | ペアで自分の理解を説明し合う |
| **Submit** | 15分 | 3点セット（コード＋考え方メモ＋I/O）を整理して提出 |

### フレームの設計意図

- **Play**: 実践から入る。講義の前にまず体験する
- **Explore**: AIの出力を「読解・比較・修正」の対象として使う
- **Trace**: 状態遷移を追えることが理解の証明
- **Explain**: 口頭説明を日常的な学習活動にする（普段はペア、主要課題では個別口頭試問）
- **Submit**: 3点セットの提出を毎回繰り返すことで型を定着させる

## 第1回: ガイダンス＋環境構築

### 概要

| 項目 | 内容 |
|------|------|
| テーマ | 環境構築 |
| 核となる問い | 「動いた？」 |
| フレーム | ガイダンス → 環境構築 → 動作確認 → 次回予告（通常フレームは使わない） |

### 授業の流れ

| Phase | 時間 | 内容 |
|-------|------|------|
| ガイダンス | 30分 | 授業の目標・ルール・評価方法の説明。「AIは使ってよい。説明できるかが問われる」を共有。使用ツールの紹介（Notion / Slack / Google Classroom） |
| 環境構築 | 60分 | 環境構築ガイドに沿ってセットアップ。グループで助け合いながら進める |
| 動作確認 | 20分 | 用意されたpy5スケッチ（円が1個表示される）を実行。色や座標の数値を1つ変えて「何が変わったか」を確認 |
| 次回予告 | 10分 | 提出物の3点セットの説明。次回の予習指示 |

### 環境構築の手順

```
Homebrew → uv → Python 3.12 → JDK (Temurin 21) → py5 → Visual Studio Code
```

### 提出物

- 環境構築完了のスクリーンショット

### 予習（次回まで）

- ターミナルの基本コマンド（ls, cd, pwd）を調べる

### ポイント

- この回はフレームを使わない。環境構築が最優先
- 最後の20分で「コードを変えて何が変わるか観察する」体験を入れ、次回からのPlayフェーズへの橋渡しにする
- グループで助け合いながらセットアップすることでPlayful Learningが始まる

## 第2回: 順次制御と図形描画

### 概要

| 項目 | 内容 |
|------|------|
| テーマ | 順次制御 |
| 核となる問い | 「順番を変えると何が変わる？」 |
| 概念 | プログラムは上から下へ1行ずつ実行される |

### 授業の流れ

| Phase | 時間 | 内容 |
|-------|------|------|
| **Play** | 30分 | 10行程度の完成コード（家や風景を描く）を実行。「何が描かれた？」をグループで観察。行の順番を入れ替えたら出力がどう変わるか実験 |
| **Explore** | 30分 | `rect()`, `circle()`, `fill()`, `stroke()` の引数を変えて試す。AIに「木を描くコード」を出させ、自分のコードと比較する |
| **Trace** | 25分 | 5行のコードを上から順に追い「3行目の時点でキャンバスに何があるか」を描く。順次実行＝上から下への概念を確認 |
| **Explain** | 20分 | ペアで「このコードの4行目は何をしているか」を説明し合う |
| **Submit** | 15分 | 3点セットを整理して提出 |

### 提出物（3点セット）

- **コード**: 自分で修正・作成した図形描画コード
- **考え方メモ**: どのような図形を描こうとしたか、なぜその順番で描いたか
- **入力と期待結果**: 「2行目と4行目を入れ替えると何が変わるか」の予測と実行結果

### 予習（次回まで）

- 変数の概念を調べる
- レポートの体裁（PDFの作り方）を確認

## 第3回: 変数と状態変化

### 概要

| 項目 | 内容 |
|------|------|
| テーマ | 変数 |
| 核となる問い | 「この行でxはいくつ？」 |
| 概念 | 変数＝名前付きの箱。代入で中身が変わる |

### 授業の流れ

| Phase | 時間 | 内容 |
|-------|------|------|
| **Play** | 30分 | 変数で図形の位置・色を制御するコードを実行。同じ図形を「変数あり」と「数値直書き」の2バージョンで見比べる |
| **Explore** | 30分 | 変数の値を変えて出力の変化を観察。AIに「変数を使って5つの円を描くコード」を出させ、自分の書き方と比較 |
| **Trace** | 25分 | 状態トレース表を書く。各行で変数の値がどう変わるか記録 |
| **Explain** | 20分 | 状態トレース表をペアで見せ合い「3行目でxが150になる理由」を説明 |
| **Submit** | 15分 | 3点セットを整理して提出 |

### 状態トレース表の例

```
行 | コード              | x   | y   | 画面の変化
1  | x = 50             | 50  | -   | なし
2  | y = 100            | 50  | 100 | なし
3  | x = x + 100        | 150 | 100 | なし
4  | circle(x, y, 30)   | 150 | 100 | (150,100)に円
5  | x = x + 100        | 250 | 100 | なし
6  | circle(x, y, 30)   | 250 | 100 | (250,100)に円
```

### 提出物（3点セット）

- **コード**: 変数を使った図形描画コード
- **考え方メモ**: 状態トレース表（各行での変数の値）
- **入力と期待結果**: 「xの初期値を変えたら何が変わるか」の予測と実行結果

### 予習（次回まで）

- for文、range()関数を調べる

## 第4回: 繰り返し処理（for文）

### 概要

| 項目 | 内容 |
|------|------|
| テーマ | 繰り返し |
| 核となる問い | 「3回目のループで何が起きる？」 |
| 概念 | 同じ処理を繰り返す。ループ変数が毎回変わる |

### 授業の流れ

| Phase | 時間 | 内容 |
|-------|------|------|
| **Play** | 30分 | AIに「円を10個横に並べるコード」を出させて実行。「for文がないバージョン」も出させて比較。「10行 vs 3行、何が違う？」 |
| **Explore** | 30分 | `range(10)` を `range(5)` や `range(20)` に変える。間隔や色を変える実験。「変える前に何が起きるか予測→実行→確認」のサイクルを回す |
| **Trace** | 25分 | ループトレース表を書く。各反復でiの値と図形の位置を記録 |
| **Explain** | 20分 | ペアで「3回目のループでは何が起きるか」を口頭で説明 |
| **Submit** | 15分 | 3点セットを整理して提出 |

### ループトレース表の例

```
反復 | i の値 | x = i * 50 | 描かれる円の位置
1    | 0     | 0          | (0, 200)
2    | 1     | 50         | (50, 200)
3    | 2     | 100        | (100, 200)
4    | 3     | 150        | (150, 200)
5    | 4     | 200        | (200, 200)
```

### 提出物（3点セット）

- **コード**: for文を使った図形描画コード
- **考え方メモ**: ループトレース表（各反復での変数の値）
- **入力と期待結果**: 「range(5)をrange(10)に変えたときの予測と実行結果」

### 予習（次回まで）

- for文の入れ子（二重ループ）を調べる

## 第5回: 多重ループとアニメーション

### 概要

| 項目 | 内容 |
|------|------|
| テーマ | 多重ループ＋アクティブモード |
| 核となる問い | 「i=1, j=2 のとき何が描かれる？」 |
| 概念 | for文の入れ子で2次元パターン。`setup()` + `draw()` でアニメーション |

### 授業の流れ

| Phase | 時間 | 内容 |
|-------|------|------|
| **Play** | 30分 | 二重ループで格子状に円が並ぶコードを実行。次に `setup()` + `draw()` を使ったアニメーション（`frame_count` で色が変わる）を実行して違いを観察 |
| **Explore** | 30分 | 格子の条件を変えて市松模様・グラデーションを作る。AIに「別のパターン」を出させて読み解く |
| **Trace** | 25分 | 二重ループで `i=1, j=2` のとき図形がどこに描かれるかを追う。`draw()` が繰り返し呼ばれる仕組みを確認 |
| **Explain** | 20分 | グループで「外側ループと内側ループの役割の違い」を説明し合う |
| **Submit** | 15分 | 3点セットを整理して提出 |

### 提出物（3点セット）

- **コード**: 二重ループまたはアニメーションのコード
- **考え方メモ**: パターンの設計意図、または `draw()` が繰り返し呼ばれることの説明
- **入力と期待結果**: 「内側ループのrangeを変えたときの予測と実行結果」

### 予習（次回まで）

- if文の構文を調べる
- py5の `mouse_x` / `mouse_y` を調べる

## 第6回: 条件分岐とインタラクション

### 概要

| 項目 | 内容 |
|------|------|
| テーマ | 条件分岐 |
| 核となる問い | 「境界はどこ？なぜそこで変わる？」 |
| 概念 | if/elif/else で処理を分ける。マウス・キーボード入力に応答する |

### 授業の流れ

| Phase | 時間 | 内容 |
|-------|------|------|
| **Play** | 30分 | マウス位置で色が変わるインタラクティブスケッチを触る。「画面のどこで色が切り替わる？境界はどこ？」をグループで探る |
| **Explore** | 30分 | 閾値（100→200）を変える実験。AIに「キーボードで形が切り替わるコード」を出させて読む。意図的にバグを入れたコードを修正する |
| **Trace** | 25分 | 分岐トレース表を書く。入力値ごとにどの経路を通り、何が起きるか記録 |
| **Explain** | 20分 | ペアで「mouse_xが150のとき、なぜこの色になるか」を説明 |
| **Submit** | 15分 | 3点セットを整理して提出 |

### 分岐トレース表の例

```
mouse_x | 条件判定              | 通る経路 | 画面の結果
50      | mouse_x < 100 → True | if     | 背景が赤
150     | mouse_x < 200 → True | elif   | 背景が青
250     | どちらもFalse          | else   | 背景が緑
```

### 提出物（3点セット）

- **コード**: 条件分岐を使ったインタラクティブなスケッチ
- **考え方メモ**: 分岐トレース表（入力値ごとの経路と結果）
- **入力と期待結果**: 複数の入力値での予測と実行結果

### 予習（次回まで）

- Pythonの関数定義（def）を調べる

## 第7回: 関数とスコープ

### 概要

| 項目 | 内容 |
|------|------|
| テーマ | 関数 |
| 核となる問い | 「なぜ関数に分けた？」 |
| 概念 | 処理をまとめて名前をつける。引数と戻り値。グローバル変数とローカル変数 |

### 授業の流れ

| Phase | 時間 | 内容 |
|-------|------|------|
| **Play** | 30分 | 関数化された作品コード（`draw_tree(x,y,size)` で木を複数描く）を実行。同じコードの「関数なし版」（コピペで書かれた版）も動かして比較 |
| **Explore** | 30分 | 引数を変えて出力の変化を確認。AIに「第5-6回のコードを関数で整理したバージョン」を出させ、元のコードと構造を比較 |
| **Trace** | 25分 | 関数呼び出し時の引数の流れを追う。グローバル変数とローカル変数の違いを「同じ名前の変数が関数の中と外にあるコード」で確認 |
| **Explain** | 20分 | グループで「なぜこの処理を関数にしたのか」を発表。設計理由を説明できることが目標 |
| **Submit** | 15分 | 3点セットを整理して提出 |

### 提出物（3点セット）

- **コード**: 関数を使って整理したコード
- **考え方メモ**: なぜその関数分割にしたか（設計理由）
- **入力と期待結果**: 「引数を(100,200,50)→(300,100,80)に変えたときの予測と実行結果」

### 予習（次回まで）

- 第1-7回の復習
- 配列（リスト）を調べる
